# 自动任务执行系统 - 简化版设计方案

> **产品定位**: 个人助手
> **设计原则**: 简洁、高效、最小实现
> **参考架构**: OpenCLAW
> **实施周期**: 1周

---

## 一、设计理念

### 1.1 产品定位

**个人助手场景特征**：
- 单用户/小团队使用
- 任务数量有限（< 100个）
- 无需分布式部署
- 重视响应速度和易用性

### 1.2 核心原则

- ✅ **保持简洁**: JSON 文件存储，无需数据库
- ✅ **内置调度**: 无需 APScheduler 等第三方调度器
- ✅ **复用架构**: 深度集成现有 ReAct Agent 系统
- ✅ **最小实现**: 仅实现核心功能，避免过度设计

### 1.3 架构对比

| 维度 | 原设计 | 简化版 | 说明 |
|------|--------|--------|------|
| **消息路由** | Gateway 服务 | EventBus | 去掉独立 Gateway |
| **调度器** | TaskScheduler | SimpleScheduler | 简化调度逻辑 |
| **cron 解析** | croniter 库 | 预设模板 | 仅支持常用格式 |
| **步骤类型** | agent_message + tool_call | agent_message | 去掉 tool_call |
| **步骤依赖** | DAG 依赖图 | 顺序执行 | 简化执行流程 |
| **重试机制** | 步骤级重试 | 任务级重试 | 简化错误处理 |
| **数据存储** | JSON 文件 | JSON 文件 | 保持不变 |
| **并发控制** | 信号量 | 简单计数 | max_concurrent=3 |
| **代码量** | ~2000 行 | ~1050 行 | **减少 47%** |

### 2.2 核心组件

#### 2.2.1 EventBus (事件总线)

**职责**: 替代 Gateway，提供简单的事件广播机制

**特点**:
- 轻量级实现（~50 行代码）
- 集成到现有 `/ws/chat` 端点
- 单向推送（服务端 → 客户端）

**实现**:
```python
class EventBus:
    def __init__(self):
        self.clients: List[WebSocket] = []

    def register(self, ws: WebSocket):
        self.clients.append(ws)

    def unregister(self, ws: WebSocket):
        if ws in self.clients:
            self.clients.remove(ws)

    async def broadcast(self, event_type: str, data: dict):
        """广播事件到所有客户端"""
        message = {
            "type": "event",
            "event": event_type,
            "data": data,
            "timestamp": datetime.now().isoformat()
        }

        dead_clients = []
        for client in self.clients:
            try:
                await client.send_json(message)
            except:
                dead_clients.append(client)

        # 清理断开的连接
        for client in dead_clients:
            self.unregister(client)
```

**支持的事件**:
- `task.started` - 任务开始执行
- `task.completed` - 任务执行完成
- `task.failed` - 任务执行失败

#### 2.2.2 SimpleScheduler (调度器)

**职责**: 任务调度和生命周期管理

**核心功能**:
1. 任务注册/启用/禁用
2. cron 调度（预设模板）
3. 手动触发
4. 并发控制（max_concurrent=3）
5. 超时保护（1小时）

**简化点**:
- ❌ 不支持复杂 cron 表达式（仅支持 3 种预设）
- ❌ 不支持任务依赖图
- ❌ 不支持健康检查循环
- ❌ 不支持自动数据清理

**预设 cron 模板**:
```python
CRON_TEMPLATES = {
    "0 8 * * *": "每天 8:00",
    "0 */2 * * *": "每 2 小时",
    "*/30 * * * *": "每 30 分钟"
}
```

**调度循环**:
```python
async def _schedule_loop(self):
    """调度循环（每分钟检查一次）"""
    while self._running:
        await asyncio.sleep(60)

        for task in self.tasks.values():
            if not task.enabled:
                continue

            if self._should_run(task):
                await self._try_execute(task)
```

#### 2.2.3 ScheduledTaskExecutor (执行器)

**职责**: 执行任务步骤

**核心功能**:
1. agent_message 模式（主要）
2. 顺序执行步骤
3. 基础错误处理

**简化点**:
- ❌ 不支持 tool_call 模式
- ❌ 不支持步骤依赖图
- ❌ 不支持步骤重试
- ❌ 不支持上下文传递

**执行流程**:
```python
async def execute(self, task: TaskTemplate) -> Dict[str, Any]:
    """执行任务"""
    results = {}

    for step in task.steps:
        try:
            result = await self._execute_step(step)
            results[step.step_id] = result

            # 如果步骤失败且不继续，则停止
            if result.get("status") == "failed" and not step.continue_on_failure:
                break

        except Exception as e:
            results[step.step_id] = {
                "status": "failed",
                "error": str(e)
            }
            if not step.continue_on_failure:
                break

    return results
```

## 三、数据模型

### 3.1 TaskTemplate (任务模板)

```python
class TaskSchedule(BaseModel):
    """调度配置"""
    type: Literal["cron", "interval"]
    cron_expression: Optional[str] = None  # 仅支持预设模板
    interval_seconds: Optional[int] = None

class TaskStep(BaseModel):
    """任务步骤"""
    step_id: str
    type: Literal["agent_message"]  # 仅支持 agent_message
    content: str  # Agent 查询内容
    continue_on_failure: bool = False

class TaskTemplate(BaseModel):
    """任务模板"""
    id: str
    name: str
    description: str
    enabled: bool = True
    schedule: TaskSchedule
    steps: List[TaskStep]
    state: Dict[str, Any] = Field(default_factory=dict)  # 运行时状态
    created_at: datetime = Field(default_factory=datetime.now)
    updated_at: datetime = Field(default_factory=datetime.now)
```

### 3.2 TaskExecution (执行记录)

```python
class TaskExecution(BaseModel):
    """任务执行记录"""

    step_results: Dict[str, Any] = Field(default_factory=dict)
    error_message: Optional[str] = None
```

### 7.1 create_scheduled_task 工具集成 (新增)

```python
# backend/app/tools/scheduled_tasks/create_task_tool.py

from app.agent.core.tool_base import BaseTool
from app.scheduler.simple_scheduler import scheduler
from app.scheduler.models import TaskTemplate, TaskSchedule, TaskStep
from pydantic import BaseModel, Field
from typing import Literal, List
import time
from datetime import datetime, timedelta

class TaskStepInput(BaseModel):
    """任务步骤输入"""
    content: str = Field(description="Agent 要执行的查询内容")
    continue_on_failure: bool = Field(default=False, description="失败后是否继续")

class CreateScheduledTaskInput(BaseModel):
    """创建定时任务输入"""
    name: str = Field(description="任务名称")
    description: str = Field(description="任务描述")
    schedule_type: Literal["daily_8am", "every_2hours", "every_30min"] = Field(
        description="调度类型：daily_8am(每天8点), every_2hours(每2小时), every_30min(每30分钟)"
    )
    steps: List[TaskStepInput] = Field(description="任务步骤列表")
    enabled: bool = Field(default=True, description="是否启用")

class CreateScheduledTaskTool(BaseTool):
    """创建定时任务工具"""

    name = "create_scheduled_task"
    description = """创建定时任务，让系统自动在指定时间执行分析任务。

使用场景：
- 每日定时生成污染报告
- 定期监控数据变化
- 自动备份重要数据

支持的调度类型：
- daily_8am: 每天早上8:00执行
- every_2hours: 每2小时执行一次
- every_30min: 每30分钟执行一次
"""

    input_schema = CreateScheduledTaskInput
    requires_context = False

    # cron 模板映射
    SCHEDULE_MAP = {
        "daily_8am": "0 8 * * *",
        "every_2hours": "0 */2 * * *",
        "every_30min": "*/30 * * * *"
    }

    async def execute(self, **kwargs) -> dict:
        """执行工具"""
        # 生成任务ID
        task_id = f"task_{int(time.time())}"

        # 转换调度类型
        cron_expr = self.SCHEDULE_MAP[kwargs["schedule_type"]]

        # 构建任务模板
        task = TaskTemplate(
            id=task_id,
            name=kwargs["name"],
            description=kwargs["description"],
            enabled=kwargs["enabled"],
            schedule=TaskSchedule(
                type="cron",
                cron_expression=cron_expr
            ),
            steps=[
                TaskStep(
                    step_id=f"step_{i+1}",
                    type="agent_message",
                    content=step.content,
                    continue_on_failure=step.continue_on_failure
                )
                for i, step in enumerate(kwargs["steps"])
            ],
            state={},
            created_at=datetime.now()
        )

        # 注册到调度器
        scheduler.tasks[task_id] = task
        scheduler._save_tasks()

        return {
            "status": "success",
            "success": True,
            "data": {
                "task_id": task_id,
                "task_name": kwargs["name"],
                "schedule": self._format_schedule(kwargs["schedule_type"]),
                "next_run": self._calculate_next_run(cron_expr)
            },
            "summary": f"已创建定时任务「{kwargs['name']}」，将在{self._format_schedule(kwargs['schedule_type'])}自动执行"
        }

    def _format_schedule(self, schedule_type: str) -> str:
        """格式化调度描述"""
        formats = {
            "daily_8am": "每天早上8:00",
            "every_2hours": "每2小时",
            "every_30min": "每30分钟"
        }
        return formats[schedule_type]

    def _calculate_next_run(self, cron_expr: str) -> str:
        """计算下次执行时间"""
        now = datetime.now()

        if cron_expr == "0 8 * * *":
            # 每天8点
            next_run = now.replace(hour=8, minute=0, second=0, microsecond=0)
            if next_run <= now:
                next_run += timedelta(days=1)
        elif cron_expr == "0 */2 * * *":
            # 每2小时
            next_hour = (now.hour // 2 + 1) * 2
            next_run = now.replace(hour=next_hour % 24, minute=0, second=0, microsecond=0)
            if next_hour >= 24:
                next_run += timedelta(days=1)
        elif cron_expr == "*/30 * * * *":
            # 每30分钟
            next_minute = (now.minute // 30 + 1) * 30
            next_run = now.replace(minute=next_minute % 60, second=0, microsecond=0)
            if next_minute >= 60:
                next_run += timedelta(hours=1)

        return next_run.strftime("%Y-%m-%d %H:%M:%S")
```

## 九、限制和约束

### 9.1 功能限制

| 功能 | 支持情况 | 说明 |
|------|----------|------|
| **cron 表达式** | 仅支持 3 种预设 | `0 8 * * *`, `0 */2 * * *`, `*/30 * * * *` |
| **步骤类型** | 仅支持 agent_message | 不支持 tool_call 模式 |
| **步骤依赖** | 顺序执行 | 不支持 DAG 依赖图 |
| **步骤重试** | 不支持 | 仅支持任务级重试（手动） |
| **并发任务** | 最多 3 个 | 避免资源耗尽 |
| **执行历史** | 保留最近 50 条 | 需手动清理旧记录 |
| **任务超时** | 固定 1 小时 | 不可配置 |
| **任务创建方式** | 对话 + 前端表单 | 两种方式均支持 |

### 8.2 性能约束

- **任务数量**: < 100 个（JSON 文件存储）
- **并发执行**: 最多 3 个任务
- **调度精度**: 1 分钟（调度循环间隔）
- **执行历史**: 最多 50 条记录

### 8.3 部署约束

- **单机部署**: 不支持分布式
- **无持久化队列**: 重启后未执行的任务丢失
- **无高可用**: 无故障转移机制
