# OBM化学机理分析工具工作手册

## 1. 概述

本项目提供OBM（Ozone Box Model，臭氧箱模型）分析工具：

| 工具 | 文件 | 功能 | 复杂度 |
|------|------|------|--------|
| calculate_obm_full_chemistry | pybox_integration/tool.py | 完整EKMA分析 + PO3 + RIR + 减排模拟 | 高（RACM2机理） |

**核心功能**：
- EKMA等浓度曲线分析（预计算加速模式，~30秒）
- PO3（臭氧生成速率）分析
- RIR（相对增量反应性）分析
- 减排情景模拟
- 判断O3生成敏感性类型（VOCs-limited/NOx-limited/transitional）

**关键特性**：
- RACM2化学机理（102物种，504反应）
- 场景感知缓存系统（城市/工业/农村典型场景）
- 相似缓存插值（命中率从<5%提升至50-70%）
- 完整ODE网格模式（21×21，441点）
- 图片缓存服务（base64 → URL渲染）

### B.3 性能瓶颈分析

#### B.3.1 为什么OBM-deliver需要56秒？

**计算量分解**:
```
总ODE步数: 130,678步
每步操作:
  1. 计算导数（f(y,t)）: 102物种 × 504反应 = ~50,000次乘法
  2. 雅可比矩阵（每0.95%步）: 102×102 = 10,404个元素（稀疏）
  3. LU分解（每26%步）: O(n³) ≈ 1,061,208次操作（n=102）

总浮点运算量估算:
- 导数计算: 130,678 × 50,000 = 6.5 × 10^9次（6.5 GFLOP）
- 雅可比矩阵: 1,235 × 10,404 = 1.3 × 10^7次
- LU分解: 34,058 × 1,061,208 = 3.6 × 10^10次（36 GFLOP）
- 总计: ~43 GFLOP

单线程Fortran性能: ~1 GFLOPS（典型CPU）
理论时间: 43 GFLOP ÷ 1 GFLOPS = 43秒
实际时间: 56秒（差异来自I/O、内存访问）
```

**瓶颈识别**:
1. **LU分解**（64%计算量）- 主要瓶颈
2. **导数计算**（15%）
3. **雅可比矩阵**（10%）
4. **磁盘I/O**（5%）
5. **其他**（6%）
