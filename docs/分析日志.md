● 分析两类EKMA曲线的显著差异，我找到了以下核心原因：

  ---
  关键差异对比

  | 特征      | PyBox Integration (左图) | 规范EKMA (右图)        |
  |---------|------------------------|--------------------|
  | 高O3区域位置 | 右下角（高VOCs+低NOx）        | 脊线区域（中等VOCs+中等NOx） |
  | NOx滴定效应 | 明显（左上角低O3）             | 非常明显（低NOx区域O3反而高）  |
  | 等值线形态   | 基本水平，缺少"L形"            | 明显L形/U形脊线结构        |
  | 坐标系     | 绝对浓度（ppb）              | 归一化排放倍数（0-2倍）      |

  ---
  造成差异的原因

  1. 模拟时长过短 ⚠️ 最关键

● Read(backend\app\tools\analysis\pybox_integration\config.py)
  ⎿  Read 10 lines

● simulation_time: float = 7200.0  # 2小时 ⚠️

  问题：规范EKMA需要6-8小时模拟完整光化学反应周期，才能观察到：
  - O3逐步累积（上午）
  - 达到峰值（下午2-3点）
  - 自然衰减（傍晚）

  2小时模拟只捕捉到O3生成的初始阶段，无法呈现完整的NOx滴定-解除-再光化学生成的复杂过程。

  ---
  2. 快速模式的泰勒展开局限性

  PyBox使用的simulate_ekma_grid_fast()方法：

  # 核心算法（见pybox_adapter.py:576-638行）
  1. 基准点模拟 (VOC=1.0, NOx=1.0) → O3_base
  2. VOC扰动模拟 (VOC=1.05) → O3_voc
  3. NOx扰动模拟 (NOx=1.05) → O3_nox
  4. 计算敏感性: dO3/dVOC, dO3/dNOx
  5. 泰勒展开外推: O3(f) ≈ O3_base + dO3/dVOC*(f-1) + dO3/dNOx*(f-1) + 修正项

  问题：
  - 一阶泰勒展开无法捕捉非线性化学机制（EKMA曲线本质是高度非线性的）
  - 物理约束修正（pybox_adapter.py:737-751行）虽有帮助，但仍是启发式拟合
  - 无法准确模拟低NOx区域的"O3反弹"效应（NOx滴定解除后O3骤升）

  ---
  3. 初始O3浓度偏低

  initial_o3 = self._calculate_initial_o3(o3_data)  # 通常30ppb（10%分位数）

  问题：规范EKMA的初始O3通常设为40-50 ppb（区域背景O3），如果起点太低：
  - 低NOx区域无法体现NOx滴定后的O3恢复效应
  - 整体O3响应曲面偏低

  ---
  4. NO/NO2分配比例

  # pybox_adapter.py:434行
  initial_conc["NO2"] = adj_nox * 0.9  # 90% NO2
  initial_conc["NO"] = adj_nox * 0.1   # 10% NO

  问题：实际大气中NO比例可能更高（20-40%），这导致：
  - NO滴定效应被低估（高NO会直接消耗O3: O3 + NO → NO2 + O2）
  - 无法准确呈现低NOx区域的U型特征

  ---
  5. 固定气象参数

  temperature=298.15,  # 固定25°C
  solar_zenith_angle=30.0,  # 固定30度

  问题：规范EKMA通常模拟变化的气象条件（温度升高、太阳辐射增强），固定参数导致：
  - 光解速率恒定（实际应随时间变化）
  - 温度依赖的反应速率失真

  ---
  优化建议

  方案A：调整关键参数（快速修复）

  # 修改config.py
  simulation_time: float = 25200.0  # 7小时（清晨6:00→下午1:00）
  initial_o3_base: float = 45.0     # 提高初始背景O3

  # 修改pybox_adapter.py的NO/NO2比例
  initial_conc["NO2"] = adj_nox * 0.7  # 70% NO2
  initial_conc["NO"] = adj_nox * 0.3   # 30% NO（增加滴定效应）

  方案B：使用完整ODE积分（精确模式）

  # 弃用快速模式，使用完整ODE
  result = adapter.simulate_ekma_grid(  # 传统方法，121次ODE积分
      base_vocs={...},
      base_nox=30.0,
      simulation_time=25200.0  # 7小时
  )
  # 缺点：计算时间30秒→2分钟（10-20秒→2分钟）

  方案C：增加采样点密度（Standard模式优化）

  ekma_full.py已实现的Standard模式理论上应该更准确（采样25个关键点），但可能需要：
  - 增加低NOx区域的采样点（捕捉U型拐点）
  - 延长simulation_time到6-8小时

  ---
  结论：为什么不一致

  | 原因      | 影响程度  | 体现                |
  |---------|-------|-------------------|
  | 模拟时长2小时 | ⭐⭐⭐⭐⭐ | 缺少完整O3生成周期，无法形成脊线 |
  | 泰勒展开外推  | ⭐⭐⭐⭐  | 无法捕捉非线性U型特征       |
  | 初始O3偏低  | ⭐⭐⭐   | 低NOx区域O3反弹不明显     |
  | NO比例偏低  | ⭐⭐    | NOx滴定效应减弱         |
  | 固定气象参数  | ⭐     | 光化学过程简化           |

  核心矛盾：PyBox追求10-20秒快速响应（用户友好），而规范EKMA需要数分钟精确计算（科研标准）。
