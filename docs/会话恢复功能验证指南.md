# 会话恢复功能验证指南

## 功能概述

实现了完整的会话恢复功能，允许用户在主对话界面恢复上次对话状态并继续提问。

## 实现的修改

### 1. 后端修改

#### session_memory.py
- 添加 `load_history_messages()` 方法，支持批量导入历史消息
- 支持两种格式：
  - ReAct 事件格式：`{"type": "thought/action/observation", "data": {...}}`
  - 标准消息格式：`{"role": "user/assistant", "content": "..."}`

#### loop.py
- `run()` 方法添加 `initial_messages` 参数
- `_run_react_loop()` 方法在添加新用户消息前注入历史消息
- 通过 `self.memory.session.load_history_messages()` 加载历史对话

#### react_agent.py
- `analyze()` 方法添加 `initial_messages` 参数
- 将参数传递给 `loop.run()`

#### agent.py
- 在加载会话时，将 `conversation_history` 传递给 `agent.analyze()`
- 添加日志记录历史消息加载过程

### 2. 前端（已有功能）

- ReactTopBar：会话恢复按钮
- ReactAnalysisView：`handleSessionRestore()` 方法加载会话历史并恢复到 store
- ReActMessageList：显示所有历史消息

## 测试步骤

### 步骤1：创建初始对话

1. 启动后端服务：`cd backend && start.bat`
2. 启动前端服务：`cd frontend && npm run dev`
3. 访问 http://localhost:5174
4. 选择"通用Agent"模式
5. 提交一个查询，例如：
   ```
   分析广州2024年6月的O3污染特征
   ```
6. 等待分析完成，记下会话ID（显示在任务进度标签页）

### 步骤2：恢复会话并继续对话

1. 刷新页面或关闭后重新打开
2. 点击顶部的"恢复会话"按钮
3. 选择之前的会话ID
4. **验证点1**：查看消息列表，应该显示之前的所有对话记录：
   - 思考过程
   - 工具调用
   - 工具结果
   - 最终答案
5. 提交新的问题，例如：
   ```
   继续分析：O3的主要来源有哪些？
   ```
6. **验证点2**：观察 Agent 的回复是否引用了之前的分析结果
7. **验证点3**：检查后端日志，应该看到：
   ```
   [INFO] initial_messages_loaded session_id=xxx message_count=4
   [INFO] passing_conversation_history_to_agent session_id=xxx message_count=4
   ```

### 步骤3：验证上下文连续性

继续提问几轮，验证 Agent 能够：
- 记住之前的分析数据
- 引用之前的结论
- 避免重复查询已有数据

## 预期行为

### 成功的会话恢复应该：

1. **前端显示**：
   - 所有历史消息正确显示
   - 消息格式正确（思考/行动/观察/完成）
   - 可视化图表（如果有）正确显示

2. **Agent 行为**：
   - 能够看到之前的对话历史
   - 提到"之前的分析"、"根据刚才的数据"等上下文引用
   - 避免重复执行已完成的工具调用

3. **日志输出**：
   ```
   [INFO] session_restored session_id=session_xxx
   [INFO] passing_conversation_history_to_agent message_count=4
   [INFO] history_messages_loaded session_id=session_xxx message_count=4
   ```

## 故障排查

### 问题1：历史消息未显示

- 检查前端 console，查看 `store.messages` 是否有数据
- 检查后端日志，确认会话已成功加载
- 验证会话文件是否存在：`backend_data_registry/sessions/{session_id}.json`

### 问题2：Agent 无法看到历史上下文

- 检查后端日志，确认 `initial_messages_loaded` 日志存在
- 检查 `loop.py` 中 `load_history_messages()` 是否被调用
- 验证 `conversation_history` 格式是否正确

### 问题3：消息格式错误

- 检查保存的消息格式，必须包含 `type` 和 `data` 字段
- 验证 `session_memory.py` 中的 `load_history_messages()` 方法是否正确处理了消息格式

## 技术细节

### 消息流转

```
前端恢复会话
  ↓
GET /api/sessions/{session_id}/restore
  ↓
加载 session.conversation_history
  ↓
前端: store.messages = conversation_history
  ↓
用户继续提问
  ↓
POST /api/analyze (带 session_id)
  ↓
agent.py: 加载 conversation_history
  ↓
analyze_kwargs["initial_messages"] = conversation_history
  ↓
agent.analyze(**analyze_kwargs)
  ↓
react_agent.analyze(initial_messages=...)
  ↓
loop.run(initial_messages=...)
  ↓
session.load_history_messages(initial_messages)
  ↓
Agent 看到完整历史上下文
```
