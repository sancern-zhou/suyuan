## 问题定位

### 根本原因

**LLM在JSON中使用了中文引号 `"报告模板"`，导致JSON解析失败！**

### 关键证据

1. **错误位置**: line 3 column 128
2. **问题字符**: `"报告模板"` 中的中文引号 `""`
3. **系统提示词**: 已明确禁止中文引号，但LLM仍然使用

### 为什么会发生？

1. **用户查询中的影响**: 用户查询"是否包括**报告模板**目录"使用了中文书名号
2. **LLM模仿输入**: LLM在生成JSON时模仿了用户的引号风格
3. **提示词约束不足**: 虽然系统提示词禁止了中文引号，但LLM仍可能模仿用户输入

## 建议的修复

### 立即修复：增强预处理

```python
def _preprocess_llm_output(self, content: str) -> str:
    """预处理LLM输出，修复常见问题"""
    if not content:
        return content

    # 强制替换所有中文标点符号
    replacements = {
        '"': '"',  # 中文左双引号
        '"': '"',  # 中文右双引号
        ''': "'",  # 中文左单引号
        ''': "'",  # 中文右单引号
        '《': '<', '》': '>',  # 中文书名号
        '【': '[', '】': ']',  # 中文方括号
    }

    for old, new in replacements.items():
        content = content.replace(old, new)

    return content
```
