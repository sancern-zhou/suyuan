# 新旧AQI标准对比分析功能需求文档

## 1. 背景与目标

### 背景
国家环境空气质量标准（AQI计算方式和城市综合指数计算方式）发生更新，现有系统数据查询接口仍基于旧标准计算统计数据。

### 目标
实现新旧两个标准下的空气质量数据对比分析功能，支持用户查询同一时期的新旧标准计算结果，并查看差异。

## 2. 功能需求

### 2.1 核心功能
- 支持基于原始监测数据（六参数）自动计算新标准统计指标
- 对比新旧标准的AQI值、AQI等级、超标情况、综合指数、城市排名
- 返回差异分析结果（绝对值变化、百分比变化、等级变化）

### 2.2 数据源
- **原始数据**：通过现有结构化查询接口获取小时/日监测数据
- **统计数据**：接口返回的旧标准统计数据（作为基准对比）

### 2.3 计算指标
| 指标 | 旧标准 | 新标准 | 对比输出 |
|------|--------|--------|----------|
| AQI值 | 基于旧标准断点计算 | 基于新标准断点计算 | 差值、变化率 |
| AQI等级 | 旧标准等级划分 | 新标准等级划分 | 是否变化 |
| 超标判定 | 旧标准阈值 | 新标准阈值 | 是否变化 |
| 综合指数 | 旧标准公式 | 新标准公式 | 差值、排名变化 |
| 城市排名 | 基于旧标准CI排序 | 基于新标准CI排序 | 排名变化 |

## 3. 技术方案

### 3.1 整体架构
```
查询层 → 数据获取层 → 计算层 → 对比层 → 输出层
```

### 3.2 核心模块

#### 模块1：标准计算器（AQICalculator）
- 支持多标准版本的IAQI计算
- 管理新旧标准的断点配置
- 计算AQI、首要污染物、空气质量等级

#### 模块2：综合指数计算器（ComprehensiveIndexCalculator）
- 实现新旧标准的综合指数计算公式
- 支持多城市排名计算

#### 模块3：对比引擎（StandardComparisonEngine）
- 聚合原始数据（小时→日）
- 并行计算新旧标准结果
- 生成差异分析报告

#### 模块4：查询工具增强
- 在现有查询工具中增加 `enable_standard_comparison` 参数
- 可选启用标准对比功能

### 3.3 数据流程
```
1. Agent接收用户查询（如"对比广州新旧标准AQI"）
2. 调用结构化查询接口获取原始监测数据
3. 标准对比引擎处理：
   - 按城市聚合数据
   - 用旧标准计算指标
   - 用新标准计算指标
   - 计算差异
4. 返回原始数据 + 对比结果给LLM
5. LLM生成对比分析报告
```

### 5.7 环境空气质量综合指数计算方法

#### 新标准（HJ 633-2024）- 带权重算法

**计算公式**:
```
Imax = MAX(ωi · Ii)       （C.2）
Isum = Σ ωi · Ii          （C.3）
```

**参数说明**:
- `Imax`: 环境空气质量最大指数
- `Isum`: 环境空气质量综合指数
- `ωi`: 污染物i的权重
- `Ii`: 污染物i的分指数

**权重设置**:
| 污染物 | 权重 (ωi) |
|--------|-----------|
| PM2.5  | 3 |
| O3     | 2 |
| NO2    | 2 |
| SO2    | 1 |
| CO     | 1 |
| PM10   | 1 |

**设计理念**: 权重反映污染物的健康影响和污染防治管理需求，PM2.5作为首要污染物权重最高，O3和NO2次之。

#### 旧标准（HJ 633-2011）- 待补充

**TODO**: 需要补充旧标准的综合指数计算公式，以便对比新旧标准差异。

#### 预期差异分析

基于新标准引入权重算法，预期变化：
1. **PM2.5影响增大**: PM2.5浓度高的城市，综合指数会显著上升
2. **O3/NO2影响提升**: 这两项污染物的作用倍增
3. **城市排名变化**: 侧重PM2.5治理效果的城市排名可能下降

### 5.8 新旧标准数据处理差异

#### 极值处理方式对比

| 污染物 | 条件 | 旧标准处理 | 新标准处理 |
|--------|------|-----------|-----------|
| SO₂ 1小时平均 | 浓度 > 800 μg/m³ | 视为无效数据 | IAQI按200计算 |
| O₃ 8小时平均 | 浓度 > 800 μg/m³ | 视为无效数据 | IAQI按300计算 |

#### 差异影响

**旧标准**: 极端高浓度情况下直接判为无效数据，可能导致某些时段AQI缺失

**新标准**: 极端高浓度情况下给出确定的最大IAQI值，确保数据完整性：
- SO₂ 1小时平均最高IAQI = 200
- O₃ 8小时平均最高IAQI = 300

#### 实现注意事项

在进行新旧标准对比时，需要特别注意：
1. 旧标准数据中可能存在因极值而被过滤的记录
2. 新标准会保留这些记录并赋予最大IAQI值
3. 对比时需处理数据集不一致的情况（记录数量差异）

### 5.2 IAQI计算公式
```
IAQI = (IAQI_hi - IAQI_lo) / (BP_hi - BP_lo) × (C - BP_lo) + IAQI_lo
```
其中：
- `C`: 污染物项目浓度值
- `BP_hi`: 与C相近的污染物浓度限值高位值
- `BP_lo`: 与C相近的污染物浓度限值低位值
- `IAQI_hi`: 对应BP_hi的空气质量分指数
- `IAQI_lo`: 对应BP_lo的空气质量分指数

### 5.3 AQI计算公式
```
AQI = max(IAQI_1, IAQI_2, ..., IAQI_6)
```
取六项污染物的IAQI最大值

### 5.7 实现策略建议

由于新标准只调整了PM2.5和PM10的断点，其他四参数保持不变，实现时可采用以下策略：

1. **复用旧数据**: SO2、NO2、O3、CO的IAQI值直接使用旧标准计算结果
2. **仅重算PM**: 只需对PM2.5和PM10按新断点重新计算IAQI
3. **性能优化**: 减少50%的计算量，提升对比效率

### 5.8 首要污染物确定规则
- IAQI最大的污染物为首要污染物
- 当AQI≤50时，不报告首要污染物
