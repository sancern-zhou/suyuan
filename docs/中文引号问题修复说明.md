# 中文引号问题修复说明

## 问题描述

从最新的后端日志中发现，LLM生成的JSON包含**中文引号**，导致解析失败：

### 错误日志
```
2026-02-08T02:42:48.403345Z [error] parsing_failed
content_preview={...用户再次请求"列出文件目录"...}
error_type=PARSING_FAILED
```

### 问题原因
LLM在生成JSON时使用了中文引号 `""` 而非英文引号 `""`，导致JSON格式无效。

## 解决方案

### json-repair 自动处理

**好消息**：json-repair库已经自动处理了这个问题！

#### 测试验证

```python

import json

# 包含中文引号的JSON
test = '{"thought": "用户请求"列出文件目录""}'

# 使用json-repair修复
repaired = repair_json(test)
# 结果: {"thought": "用户请求\"列出文件目录\""}

# 验证可以解析
data = json.loads(repaired)  # ✅ 成功
```

## 技术细节

### json-repair 如何处理中文引号

json-repair库内部会：
1. 检测到中文引号 `""` 和 `''`
2. 自动替换为英文引号 `""` 和 `''`
3. 同时修复其他JSON格式问题（反斜杠、括号等）

### 预处理函数（备用）

虽然json-repair已经处理了中文引号，但我们还添加了预处理函数作为额外保护：

**文件**: `app/utils/llm_response_parser.py`

```python
def _preprocess_llm_output(self, content: str) -> str:
    """预处理LLM输出，修复常见问题"""
    if not content:
        return content

    # 修复中文引号
    content = content.replace('"', '"').replace('"', '"')
    content = content.replace(''', "'").replace(''', "'")

    return content
```

**调用顺序**:
1. `_preprocess_llm_output()` - 预处理（修复中文引号等）
2. `_repair_json_with_library()` - json-repair深度修复
3. 其他解析策略 - 备用方案
