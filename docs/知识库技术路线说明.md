# 知识库问答系统技术路线说明文档

## 1. 系统架构概述

### 1.1 整体架构

知识库问答系统采用 RAG（Retrieval-Augmented Generation）架构，结合向量检索与大语言模型生成能力。系统分为五个核心层次：

```
+------------------+     +------------------+     +------------------+
|    前端交互层     |     |    后端服务层     |     |    数据存储层     |
|  Vue 3 + SSE    | --> |  FastAPI + 异步   | --> |  PostgreSQL     |
|  知识问答/管理    |     |  检索/生成服务    |     |  ChromaDB       |
+------------------+     +------------------+     +------------------+
                              |
                      +-------v--------+
                      |    缓存层       |
                      | Redis + LRU    |
                      +----------------+
```

### 1.2 核心组件

| 组件 | 职责 | 技术选型 |
|------|------|----------|
| 文档处理引擎 | 解析、清洗、智能分块 | Python + LlamaIndex + 本地LLM |
| 向量存储系统 | 存储文档Embedding | ChromaDB |
| 检索服务 | 混合检索、重排序 | 向量相似度 + BM25 + Reranker |
| 问答引擎 | 对话管理、流式响应 | HyDE + SSE |
| 前端组件 | 交互界面、状态管理 | Vue 3 + Pinia + SSE |

## 2.1.1 PDF文档解析（原生 + 扫描件双模式）

PDF文档是知识库中最常见的格式，系统采用**智能检测 + 自适应处理**策略：

```
上传PDF --> 检测文本层 --> [有文本] 使用fast策略提取 --> [无文本/失败] 触发OCR
                         |                                           |
                         v                                           v
                   Unstructured解析                           PDF转图片(Qwen3-VL)
                   + GMFT表格提取                                    |
                         |                                           |
                         v                                           v
                   返回文本内容                                  返回OCR文本
```

**检测流程（_try_fast_pdf_parse）：**

```python
def _try_fast_pdf_parse(self, file_path: str) -> tuple:
    """
    尝试使用fast策略解析PDF

    Returns:
        (content, is_scanned): 内容和是否为扫描件标识
    """
    # 1. 使用Unstructured的fast策略提取文本
    elements = partition(
        filename=file_path,
        strategy="fast",  # 快速策略，优先提取文本层
        include_page_breaks=False,
        languages=["chi_sim", "eng"]  # 中英文混合
    )

    # 2. 检查是否提取到有效内容
    text_content = "".join(str(el) for el in elements).strip()

    if not text_content and not tables_text:
        # 没有文本也没有表格 → 判定为扫描件
        return ("", True)

    return (content, False)
```

#### 2.1.1.2 扫描件PDF处理（OCR模式）

当检测到PDF为扫描件（无文本层或解析失败）时，系统自动触发OCR流程：

**OCR处理架构：**

```
扫描件PDF
    |
    +---> 1. PDF转图片 (PyMuPDF)
    |          |
    |          v
    |     [页面1图片, 页面2图片, ...]
    |              |
    |              v
    +---> 2. 并发OCR调用 (Qwen3-VL多模态模型)
               |
               +---> semaphore限制并发数(默认2)
               |
               v
         [页面1文本, 页面2文本, ...]
               |
               v
         3. 结果合并，添加页码标记
               |
               v
         完整文本内容
```

**PDF转图片实现：**

```python
def _pdf_to_images(self, file_path: str, scale: float = 2.0) -> List[tuple]:
    """
    将PDF每页转换为图片

    Args:
        file_path: PDF文件路径
        scale: 缩放比例，默认2x以提高OCR精度

    Returns:
        [(page_num, image_bytes), ...]
    """
    import fitz  # PyMuPDF

    images = []
    doc = fitz.open(file_path)

    for page_num in range(len(doc)):
        page = doc[page_num]
        # 缩放矩阵：2x提升OCR分辨率
        mat = fitz.Matrix(scale, scale)
        pix = page.get_pixmap(matrix=mat)
        img_bytes = pix.tobytes("png")
        images.append((page_num, img_bytes))

    doc.close()
    return images
```

```python
async def _process_pdf_with_ocr(self, file_path: str) -> str:
    """扫描件PDF完整OCR流程"""

    # 1. PDF转图片
    images = self._pdf_to_images(file_path)

    # 2. 使用信号量限制并发（避免API限流）
    semaphore = asyncio.Semaphore(self.OCR_MAX_CONCURRENT)  # 默认2

    # 3. 并发调用OCR
    tasks = [
        process_with_semaphore(page_num, img_bytes)
        for page_num, img_bytes in images
    ]
    results = await asyncio.gather(*tasks)

    # 4. 按页码排序合并
    results.sort(key=lambda x: x[0])

    # 5. 添加页码标记，便于追溯来源
    content_parts = []
    for page_num, text in results:
        if text and not text.startswith("[OCR"):
            content_parts.append(f"<!-- 第{page_num + 1}页 -->\n{text}")

    return "\n\n".join(content_parts)
```

#### 2.1.1.3 表格提取（原生 + 扫描件通用）

系统使用**GMFT（Microsoft Table Transformer）**进行表格提取，该方法对原生PDF和OCR结果都适用：

```python
def _extract_tables_with_gmft(self, file_path: str) -> List[str]:
    """使用GMFT提取PDF表格"""

    # 初始化表格检测器和格式化器
    if not hasattr(self, '_gmft_detector'):
        self._gmft_detector = TableDetector()
        self._gmft_formatter = AutoTableFormatter()

    tables_text = []
    doc = PyPDFium2Document(file_path)

    for page_idx, page in enumerate(doc):
        # 检测表格区域
        tables = self._gmft_detector.extract(page)

        for table in tables:
            # 格式化表格为DataFrame
            ft = self._gmft_formatter.extract(table)
            df = ft.df()

            # 过滤误识别（过宽表格、只有1行的表格）
            if len(df.columns) > 10 or len(df) < 2:
                continue

            # 转换为Markdown格式
            rows = []
            # 表头
            header = ' | '.join(str(col).replace('\n', ' ').strip() for col in df.columns)
            rows.append(header)
            # 数据行
            for _, row in df.iterrows():
                cells = []
                for val in row:
                    if val is None or (isinstance(val, float) and str(val) == 'nan'):
                        cells.append("")
                    else:
                        cells.append(str(val).replace('\n', ' ').strip())
                rows.append(' | '.join(cells))

            table_text = '\n'.join(rows)
            tables_text.append(f"[表格 - 第{page_idx+1}页]\n{table_text}")

    return tables_text
```

**表格格式输出示例：**

```
[表格 - 第3页]
污染物 | 限值 | 单位
SO2 | 50 | mg/m3
NO2 | 100 | mg/m3
PM2.5 | 35 | mg/m3
```

**备用方案（pdfplumber）：** 当GMFT不可用时，自动降级为pdfplumber进行表格提取。

### 2.2 内容预处理

提取的原始内容需要经过多轮清洗，确保后续分块和检索的质量：

```python
def _preprocess_content(self, content: str) -> str:
    """预处理文档内容"""
    # 1. 统一换行，移除控制字符
    content = content.replace("\r", "")
    content = re.sub(r'[\x00-\x08\x0b\x0c\x0e-\x1f]', ' ', content)

    # 2. 移除页码标记
    content = re.sub(r'<!--\s*第\d+页\s*-->', '', content)
    content = re.sub(r'^\s*[-—]\s*\d+\s*[-—]\s*$', '', content, flags=re.MULTILINE)

    # 3. 移除目录内容（检测后跳过）
    if self._is_toc_content(content):
        return ""

    # 4. 移除连续空行
    content = re.sub(r'\n{3,}', '\n\n', content)

    return content.strip()
```

**预处理规则清单：**

| 规则 | 正则/逻辑 | 说明 |
|------|-----------|------|
| 控制字符清理 | `[\x00-\x08\x0b\x0c\x0e-\x1f]` | 移除ASCII控制字符 |
| 换行统一 | `replace("\r", "")` | 处理Windows换行符 |
| 不间断空格 | `\u00a0 -> " "` | 统一为空格 |
| 页码标记 | `<!-- 第N页 -->` | 移除OCR添加的页码 |
| 页码行 | `^\s*[-—]\s*\d+\s*[-—]\s*$` | 移除独立页码行 |
| 目录检测 | 60%以上行含`...`或页码 | 自动跳过目录页 |
| 连续空行 | `\n{3,} -> \n\n` | 合并多余空行 |
| 反斜杠转义 | `\\ -> \\\\` | 防止JSON序列化失败 |

### 2.4 分块增强

为每个分块添加上下文前缀，提升检索和生成的准确性：

```python
def _enhance_chunks_with_context_prefix(
    self,
    chunks: List[Dict[str, Any]],
    filename: str
) -> List[Dict[str, Any]]:
    """为分块添加上下文前缀"""
    enhanced_chunks = []

    for chunk in chunks:
        context_parts = []
        metadata = chunk.get("metadata", {})

        # 来源
        title = metadata.get("title", "") or Path(filename).stem
        if title:
            context_parts.append(f"来源: {title}")

        # 主题
        topic = metadata.get("topic", "")
        if topic:
            context_parts.append(f"主题: {topic}")

        # 类型
        type_map = {"paragraph": "正文", "table": "表格", "list": "列表"}
        chunk_type = metadata.get("type", "paragraph")
        context_parts.append(f"类型: {type_map.get(chunk_type, chunk_type)}")

        # 构建前缀
        if context_parts:
            context_prefix = f"[{' | '.join(context_parts)}]\n"
            enhanced_chunk = chunk.copy()
            enhanced_chunk["content"] = context_prefix + chunk["content"]
            enhanced_chunks.append(enhanced_chunk)

    return enhanced_chunks
```

```python
async def _embed_chunks(self, chunks: List[Dict[str, Any]]) -> List[Dict[str, Any]]:
    """将文本分块向量化"""
    embedding_model = self._get_embedding_model()

    for chunk in chunks:
        text = chunk["content"]
        embedding = await embedding_model.embed(text)

        chunk["vector"] = embedding
        chunk["vector_id"] = str(uuid.uuid4())
```

### 3.1 ChromaDB集成

系统使用ChromaDB作为向量数据库，支持持久化存储和高效检索：

```python
# backend/app/knowledge_base/vector_store.py

class VectorStore:
    def __init__(self, persist_directory: str = "./data/chromadb"):
        self.client = chromadb.PersistentClient(path=persist_directory)
        self.collections = {}

    def get_collection(self, kb_id: str) -> chromadb.Collection:
        """获取或创建知识库的集合"""
        if kb_id not in self.collections:
            self.collections[kb_id] = self.client.get_or_create_collection(
                name=f"kb_{kb_id}",
                metadata={"description": f"Knowledge base {kb_id}"}
            )
        return self.collections[kb_id]

    def add_chunks(self, kb_id: str, chunks: List[Dict[str, Any]]):
        """添加分块到向量库"""
        collection = self.get_collection(kb_id)

        ids = [chunk["vector_id"] for chunk in chunks]
        documents = [chunk["content"] for chunk in chunks]
        metadatas = [chunk["metadata"] for chunk in chunks]
        embeddings = [chunk["vector"] for chunk in chunks]

        collection.add(
            ids=ids,
            documents=documents,
            metadatas=metadatas,
            embeddings=embeddings
        )

    def search(
        self,
        kb_id: str,
        query_vector: List[float],
        top_k: int = 10,
        filters: Optional[Dict] = None
    ) -> List[Dict[str, Any]]:
        """向量相似度检索"""
        collection = self.get_collection(kb_id)

        results = collection.query(
            query_embeddings=[query_vector],
            n_results=top_k,
            where=filters
        )

        return self._format_search_results(results)
```

## 4. 检索系统设计

### 4.1 混合检索架构

系统采用向量检索 + 关键词检索 + 重排序的三阶段检索模式：

```
用户查询 --> 1.向量化 --> 2.向量检索(top 50) --> 3.关键词检索 --> 4.Reranker精排 --> 5.返回结果
                 |                     |                   |
             Embedding            ChromaDB            BAAI/reranker
```

**混合检索实现（service.py）：**

```python
async def search(
    self,
    request: SearchRequest
) -> SearchResponse:
    # 1. 生成查询向量
    query_vector = await self._get_query_embedding(request.query)

    # 2. 向量检索（粗筛）
    vector_results = await self._vector_search(
        query_vector=request_vector,
        kb_ids=request.kb_ids,
        top_k=50
    )

    # 3. 关键词检索（补充）
    keyword_results = await self._keyword_search(
        query=request.query,
        kb_ids=request.kb_ids,
        top_k=20
    )

    # 4. 结果融合与去重
    merged_results = self._merge_results(
        vector_results,
        keyword_results,
        weights=[0.7, 0.3]
    )

    # 5. Reranker精排
    reranked_results = await self._rerank(
        query=request.query,
        candidates=merged_results,
        top_k=request.top_k
    )

    return SearchResponse(results=reranked_results)
```

### 4.3 重排序模型

使用BAAI/reranker模型对候选结果进行精排：

```python
async def _rerank(
    self,
    query: str,
    candidates: List[Dict[str, Any]],
    top_k: int = 5
) -> List[Dict[str, Any]]:
    """使用Reranker模型精排"""
    if not candidates:
        return []

    # 准备输入
    pairs = [(query, doc["content"]) for doc in candidates]

    # 调用Reranker API
    scores = await self._rerank_model.predict(pairs)

    # 按分数排序
    for doc, score in zip(candidates, scores):
        doc["rerank_score"] = score

    candidates.sort(key=lambda x: x["rerank_score"], reverse=True)

    return candidates[:top_k]
```

### 4.4 权限过滤

检索结果需要经过权限过滤，确保用户只能访问授权的知识库：

```python
async def _filter_by_permissions(
    self,
    results: List[Dict[str, Any]],
    user_id: int,
    filters: Optional[Dict] = None
) -> List[Dict[str, Any]]:
    """过滤用户无权访问的结果"""
    allowed_kb_ids = await self._get_user_allowed_kb_ids(user_id)

    filtered = []
    for result in results:
        kb_id = result["metadata"]["kb_id"]
        if kb_id in allowed_kb_ids:
            filtered.append(result)

    return filtered
```

### 5.1 完整问答流程

```
用户问题 --> 权限检查 --> HyDE关键词生成 --> 混合检索 --> 结果精排 --> 上下文构建 --> LLM生成 --> 流式响应 --> 对话存储
```

**问答路由实现（knowledge_qa.py）：**

```python
@router.post("/chat")
async def chat(
    request: ChatRequest,
    current_user: User = Depends(get_current_user)
):
    """知识库问答接口"""
    session_id = request.session_id or str(uuid.uuid4())

    # 1. 权限验证
    if not await _check_kb_permission(request.kb_ids, current_user.id):
        raise HTTPException(status_code=403, detail="无权访问指定知识库")

    # 2. 检索相关文档

        query=request.query,
        kb_ids=request.kb_ids,
        top_k=5
    )

    # 3. 构建上下文

    # 4. 构建对话历史
    messages = _build_conversation_history(
        history=request.history,
        current_query=request.query,
        context=context
    )

    # 5. 流式生成回复
    async def generate_stream():
        async for chunk in llm_stream_generate(messages):
            yield chunk

        generate_stream(),
        media_type="text/event-stream"
```

### 5.2 上下文构建

将从知识库检索到的文档片段构建成LLM可理解的上下文：

```python
def _build_context(search_results: List[Dict[str, Any]]) -> str:
    """构建检索上下文"""
    context_parts = []

    for i, result in enumerate(search_results, 1):
        content = result["content"]
        source = result["metadata"].get("source", "未知来源")
        score = result.get("rerank_score", 0)

        context_parts.append(
            f"[文档{i}] 来源：{source} (相关度：{score:.2f})\n"
            f"内容：{content}\n"
            f"---"
        )

    return "\n\n".join(context_parts)
```

## 7. 缓存策略

### 7.1 多级缓存架构

系统实现三级缓存体系，提升检索性能：

```
L1: 内存缓存 (LRU)     -->  最快，容量有限
L2: Redis缓存          -->  较快，容量大，支持分布式
L3: Embedding缓存      -->  查询向量复用
```

**缓存实现（layered_cache.py）：**

```python
class LayeredCache:
    def __init__(self, redis_url: str = None):
        # L1: 内存缓存
        self.l1_cache = LRUCache(max_size=1000)

        # L2: Redis缓存
        self.l2_cache = None
        if redis_url:
            self.l2_cache = RedisCache(redis_url)

    async def get(self, key: str) -> Optional[Any]:
        # 1. 尝试L1
        if key in self.l1_cache:
            return self.l1_cache[key]

        # 2. 尝试L2
        if self.l2_cache:
            value = await self.l2_cache.get(key)
            if value:
                self.l1_cache[key] = value
                return value

        return None

    async def set(self, key: str, value: Any, ttl: int = 3600):
        # 同时写入L1和L2
        self.l1_cache[key] = value
        if self.l2_cache:
            await self.l2_cache.set(key, value, ttl)
```

### 7.2 缓存内容类型

| 缓存类型 | 键格式 | TTL | 用途 |
|----------|--------|-----|------|
| Embedding向量 | `emb:{query_hash}` | 24h | 查询向量复用 |
| 检索结果 | `search:{kb_ids}:{query_hash}` | 1h | 相同查询结果复用 |
| 文档分块 | `chunk:{doc_id}:{chunk_idx}` | 7d | 文档处理结果缓存 |
| 权限列表 | `perms:{user_id}` | 10min | 用户权限快速校验 |

### 7.3 缓存失效策略

- **TTL过期**：按设定时间自动过期
- **主动失效**：知识库更新时清除相关缓存
- **容量淘汰**：LRU策略淘汰最久未使用条目

```python
class KnowledgeBasePermission:
    """知识库权限管理"""

    # 权限类型
    PERMISSION_TYPES = ["read", "write", "admin"]

    async def check_permission(
        self,
        user_id: int,
        kb_id: str,
        required_level: str = "read"
    ) -> bool:
        """检查用户是否有指定权限"""

        # 1. 检查是否是知识库所有者
        is_owner = await self._check_is_owner(user_id, kb_id)
        if is_owner:
            return True

        # 2. 检查是否在共享名单中
        is_shared = await self._check_shared(user_id, kb_id)
        if is_shared:
            return True

        # 3. 检查是否是公共知识库
        is_public = await self._check_is_public(kb_id)
        if is_public and required_level == "read":
            return True

        return False
```

## 14. 常见问题

### Q1: 检索结果不相关怎么办？

1. 检查文档质量，是否有充足的上下文信息
2. 调整分块策略，使用LLM智能分块
3. 增加Reranker精排步骤
4. 调整检索权重配比

### Q2: 大文件处理慢怎么办？

1. 使用异步处理队列，不阻塞主流程
2. 优化分块策略，减少重复处理
3. 增加缓存机制，避免重复处理

### Q3: 如何支持多知识库检索？

1. 在检索时传入多个kb_ids
2. 向量库按kb_id分集合存储
3. 检索时并行查询多个集合
4. 结果合并后统一排序

### Q4: 如何实现增量更新？

1. 监听文档变更事件
2. 只重新处理变更的文档
3. 更新对应的向量记录
4. 清除相关缓存
