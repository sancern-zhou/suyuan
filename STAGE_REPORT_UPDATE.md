## 阶段性报告生成规则

### 报告累积策略
```
任务1完成 → 生成章节1（站点信息）
         ↓
任务2完成 → 追加章节2（气象条件分析）
         ↓
任务3完成 → 追加章节3（轨迹分析）
         ↓
...
         ↓
任务N完成 → 汇总所有章节 + 综合结论 → 完整报告
```

### 关键特性

1. **逐步构建**
   - 每个任务完成后立即生成对应的MD格式章节
   - 将新章节追加到已有报告内容之后
   - 确保章节之间的逻辑连贯性

2. **实时展示**
   - 每个阶段完成后向用户展示当前累积的报告内容
   - 显示报告进度（如：当前报告进度：2/7 章节完成）
   - 用户可以随时查看已生成的报告内容

3. **避免丢失**
   - 重要结论必须在对应章节中体现
   - 关键图表必须嵌入到对应章节
   - 每个数据/图表都要有专业解读

4. **格式统一**
   - 所有章节使用统一的Markdown格式
   - 标题层级清晰（一级标题、二级标题等）
   - 使用分隔符（---）区分不同章节

## 报告内容要求

### 数据支撑
- 每个结论必须有数据支撑
- 引用具体的数据值和图表
- 标注数据来源和单位

### 图表嵌入
- 关键图表必须嵌入到对应章节
- 使用Markdown图片语法：`![图表标题](image_url)`
- 每个图表都要有1-2句专业解读

### 专业解读
- 每个数据/图表都要有分析结论
- 说明数据展示的现象和问题
- 给出专业的判断和建议

### 避免丢失
- 重要结论必须在最终报告中体现
- 关键数据不能遗漏
- 确保逻辑链条完整

## 优势

### 1. 避免信息丢失
- 每个阶段的分析结论立即固化
- 不会因为后续任务而忘记前面的结论
- 累积式构建确保完整性

### 2. 用户可见
- 用户可以实时查看报告进度
- 随时了解已生成的内容
- 增强用户信任感

### 3. 易于调整
- 如果某个任务失败，已生成的报告仍然保留
- 可以基于已有内容继续分析
- 支持增量式完善报告

### 4. 格式统一
- 所有章节使用统一的Markdown格式
- 便于阅读和分享
- 支持导出为PDF等格式
