# 问数模式记忆加载修复报告

**修复日期**: 2026-04-05
**问题类型**: 记忆系统路径错误导致记忆增强失败

---

## 问题描述

问数模式（query模式）在未登录状态下无法自动加载长期记忆，导致记忆增强功能失效。

### 症状

- 用户问数时，系统无法从 `backend_data_registry/memory/query/MEMORY.md` 加载记忆
- 日志显示 `longterm_memory_disabled=True`
- 即使记忆文件有内容，也无法传递给LLM

### 日志分析

```
query_processing [app.agent.core.loop] longterm_memory_disabled=True
query_separation_verified [app.agent.core.loop] has_memory_enhancement=False
```

---

## 根因分析

### 1. 记忆路径计算错误

**文件**: `backend/app/agent/memory/memory_store.py:109-112`

**原代码**:
```python
# 全局目录：使用 {mode}_global_shared 目录
global_dir = mode_dir / f"{mode}_global_shared"
global_dir.mkdir(parents=True, exist_ok=True)
return global_dir
```

**问题**:
- 当 `user_id = "global"` 时（未登录用户），系统创建子目录 `query_global_shared`
- 实际记忆文件在 `backend_data_registry/memory/query/MEMORY.md`
- 导致读取路径: `backend_data_registry/memory/query/query_global_shared/MEMORY.md`（不存在）
- 实际文件路径: `backend_data_registry/memory/query/MEMORY.md`（存在但不匹配）

### 2. 记忆增强触发条件

**文件**: `backend/app/agent/react_agent.py:169`

```python
if self.enable_memory and manual_mode:
    # ... 加载记忆上下文
    memory_context = memory_store.get_memory_context()
    if memory_context:
        enhanced_query = f"{memory_info}\n\n用户问题：{enhanced_query}"
```

**流程**:
1. `enable_memory = True`（默认启用）
2. `manual_mode = "query"`（从API请求传递）
3. `unified_user_id = "global"`（未登录用户）
4. 调用 `memory_store.get_memory_context()`
5. 由于路径错误，读取的是空文件（新创建的模板）
6. `memory_context` 为空，记忆增强被跳过

---

## 解决方案

### 修改文件

**文件**: `backend/app/agent/memory/memory_store.py`

**修改位置**: 第109-111行

**修改前**:
```python
# 全局目录：使用 {mode}_global_shared 目录
global_dir = mode_dir / f"{mode}_global_shared"
global_dir.mkdir(parents=True, exist_ok=True)
return global_dir
```

**修改后**:
```python
# 全局目录：直接使用模式目录（backend_data_registry/memory/{mode}/）
# 例如：backend_data_registry/memory/query/MEMORY.md
return mode_dir
```

### 修改说明

1. **路径简化**: 全局用户直接使用模式目录，不再创建子目录
2. **符合直觉**: `backend_data_registry/memory/{mode}/MEMORY.md` 更容易理解
3. **向后兼容**: 与 `CLAUDE.md` 中描述的路径一致
