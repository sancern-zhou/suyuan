# 双模式架构集成测试指南

## 实施总结

### 后端实现 ✅
1. **工具注册表** (`backend/app/agent/prompts/tool_registry.py`)
   - Assistant模式：12个工具（办公任务）
   - Expert模式：22个工具（数据分析）

2. **CallSubAgentTool** (`backend/app/tools/agent_tools/call_sub_agent.py`)
   - 支持双向调用（assistant ↔ expert）
   - 继承LLMTool基类
   - 完整错误处理

3. **模式专用提示词**
   - `backend/app/agent/prompts/assistant_prompt.py` - 助手模式提示词
   - `backend/app/agent/prompts/expert_prompt.py` - 专家模式提示词

4. **提示词构建器** (`backend/app/agent/prompts/prompt_builder.py`)
   - 动态根据模式构建系统提示词

5. **ReActLoop支持** (`backend/app/agent/core/loop.py`)
   - 添加 `manual_mode` 参数

6. **API接口** (`backend/app/routers/agent.py`)
   - `AgentAnalyzeRequest` 添加 `mode` 字段（默认 expert）

7. **ReActAgent** (`backend/app/agent/react_agent.py`)
   - `analyze()` 方法支持 `manual_mode` 参数

### 前端实现 ✅
1. **模式选择器组件** (`frontend/src/components/AgentModeSelector.vue`)
   - 双按钮设计（助手/专家）
   - 图标 + 文字标签
   - 悬停提示（tooltip）
   - 渐变色高亮选中状态

2. **Store状态管理** (`frontend/src/stores/reactStore.js`)
   - 添加 `agentMode` 状态（默认 expert）
   - `startAnalysis` 方法接收 `agentMode` 参数

3. **API客户端** (`frontend/src/services/reactApi.js`)
   - `analyze` 方法添加 `agentMode` 参数
   - 请求体添加 `mode` 字段

4. **输入框组件** (`frontend/src/components/InputBox.vue`)
   - 集成 `AgentModeSelector` 组件
   - 仅在通用Agent模式（`general-agent`）显示
   - 触发 `update:agentMode` 事件

5. **主视图** (`frontend/src/views/ReactAnalysisView.vue`)
   - 处理 `agentMode` 变化事件
   - `handleSend` 方法接收并传递 `agentMode`

## 使用方式

### 1. 启动后端服务
```bash
cd backend
python -m uvicorn app.main:app --reload
```

### 2. 启动前端服务
```bash
cd frontend
npm run dev
```

### 3. 前端使用流程

1. **打开通用Agent模式**
   - 在左侧边栏选择"通用助手"或"general-agent"

2. **选择Agent模式**
   - 在输入框上方会出现"Agent模式"选择器
   - 点击"助手"按钮 → 切换到助手模式（办公任务）
   - 点击"专家"按钮 → 切换到专家模式（数据分析）

3. **输入查询**
   - **助手模式示例**：
     - "帮我读取 D:/report.txt 文件内容"
     - "生成一个污染分析报告的Word文档"
     - "执行Shell命令查看系统信息"

   - **专家模式示例**：
     - "查询广州2024-01-01的PM2.5数据"
     - "分析上风向企业污染贡献"
     - "生成空气质量趋势图"

4. **模式切换与子Agent调用**
   - 主Agent可以随时调用另一个模式作为子Agent
   - 例如：专家模式分析完数据后，可以调用助手模式生成Word报告
   - 调用示例：
     ```
     [Expert模式] 分析污染数据 → 调用 call_sub_agent(target_mode="assistant", task="生成Word报告")
     ```

## 测试场景

### 场景1：助手模式 → 专家模式
```
用户选择: 助手模式
用户输入: "帮我读取污染数据文件，然后分析PM2.5趋势"

预期流程:
1. 助手Agent读取文件
2. 助手Agent调用 call_sub_agent(target_mode="expert", task="分析PM2.5趋势")
3. 专家Agent执行数据分析
4. 返回分析结果给助手Agent
5. 助手Agent整理并回复用户
```

### 场景2：专家模式 → 助手模式
```
用户选择: 专家模式
用户输入: "分析广州O3污染原因，并生成Word报告"

预期流程:
1. 专家Agent查询数据
2. 专家Agent执行PMF/OBM分析
3. 专家Agent调用 call_sub_agent(target_mode="assistant", task="生成Word报告")
4. 助手Agent创建Word文档
5. 返回文档路径给专家Agent
6. 专家Agent整理并回复用户
```

```

```

### 场景4：纯专家模式

## 已知限制

1. **模式选择器仅在通用Agent模式显示**
   - 快速溯源、深度溯源等专用模式不显示选择器
   - 这些模式自动使用expert模式

2. **子Agent调用深度限制**
   - 子Agent最多执行10次迭代
   - 避免无限递归调用

3. **会话隔离**
   - 主Agent和子Agent共享会话记忆
   - 但子Agent的工具调用记录独立

## 故障排查

### 问题1：模式选择器不显示
**原因**：未选择通用Agent模式
**解决**：确保左侧边栏选择了"通用助手"

### 问题2：模式切换无效
**原因**：事件处理未正确绑定
**解决**：检查浏览器控制台是否有错误

### 问题3：后端收不到mode参数
**原因**：前端API调用未传递mode
**解决**：检查 `reactApi.js` 的 `body` 是否包含 `mode` 字段

### 问题4：call_sub_agent工具调用失败
**原因**：循环导入或工具未注册
**解决**：检查工具注册日志，确保 `call_sub_agent` 在工具列表中
