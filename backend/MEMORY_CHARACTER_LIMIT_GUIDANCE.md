# 记忆字符限制管理 - Agent提示词增强

## 问题背景

用户发现记忆工具内部有字符上限限制（3000字符），但记忆整合Agent不知道这个限制，可能导致：
1. Agent尝试添加记忆时失败（工具返回"已满"错误）
2. Agent不知道需要先删除旧内容
3. 降低记忆整合效率

## 限制配置

**工具限制**：`backend/app/tools/social/remember_fact/tool.py`
```python
max_size = 3000  # 记忆文件上限：3000字符

if current_size >= max_size:
    return {
        "success": False,
        "error": f"MEMORY.md已满（{current_size}/{max_size}字符），请先使用remove_memory清理旧内容",
        "summary": "记忆添加失败：记忆文件已满"
    }
```

## 解决方案：在提示词中告知Agent

### 提示词增强

**位置**：`backend/app/agent/react_agent.py:_build_consolidation_prompt()`

**新增信息**：

1. **当前记忆大小**
   ```
   **当前记忆大小**：(156/3000字符，使用率5.2%)
   ```

2. **重要限制**
   ```
   **⚠️ 重要限制**：
   - 记忆文件上限：3000字符
   - 当前使用：156字符（5.2%）
   - 当接近上限（>80%）时，优先使用 remove_memory 删除旧内容
   - 当已满（100%）时，必须先删除才能添加新内容
   ```

3. **记忆管理策略**
   ```
   **⚠️ 记忆管理策略**（基于字符限制）：
   - 当记忆使用率 < 80%：可以正常添加新记忆
   - 当记忆使用率 >= 80%：优先删除临时、过时或低优先级的记忆
   - 当记忆使用率 = 100%：必须先删除才能添加（工具会返回错误）
   ```

## Agent行为指导

### 使用率 < 80% (2400字符)
```
示例：当前1500/3000字符（50%）

策略：正常添加新记忆
- 直接使用 remember_fact 添加新信息
- 如有重复，使用 replace_memory 更新
```

### 使用率 80-99% (2400-2990字符)
```
示例：当前2560/3000字符（85.3%）

策略：谨慎添加，优先删除
- 优先删除临时信息（环境信息、过时结论）
- 保留高价值信息（用户偏好、领域知识）
- 使用 remove_memory 删除后，再添加新记忆
```

### 使用率 = 100% (3000字符)
```
示例：当前3000/3000字符（100%）

策略：必须先删除
- 工具会返回"MEMORY.md已满"错误
- 必须先使用 remove_memory 删除旧内容
- 删除足够的内容后，再尝试添加
```

## 实现细节

### 字符数计算

```python
# 获取现有记忆内容
existing_memory = memory_store.read_long_term()

# 计算字符数
current_size = len(existing_memory)
max_size = 3000  # 与工具中的限制一致
size_info = f"（{current_size}/{max_size}字符，使用率{current_size/max_size*100:.1f}%）"
```

### 提示词构建

```python
if existing_memory and existing_memory.strip():
    prompt_parts.extend([
        "## 现有记忆",
        f"**当前记忆大小**：{size_info}",
        "",
        "```",
        existing_memory,  # 完整记忆
        "```",
        "",
        "**⚠️ 重要限制**：",
        f"- 记忆文件上限：{max_size}字符",
        f"- 当前使用：{current_size}字符（{current_size/max_size*100:.1f}%）",
        "- 当接近上限（>80%）时，优先使用 remove_memory 删除旧内容",
        "- 当已满（100%）时，必须先删除才能添加新内容",
        ""
    ])
```

## 优先级指导

提示词中明确了信息优先级：
```
优先保留高价值信息（用户偏好 > 领域知识 > 历史结论 > 环境信息）
```

**删除优先级**（从高到低）：
1. **环境信息**（临时、会话相关）
2. **历史结论**（可能过时）
3. **领域知识**（相对稳定）
4. **用户偏好**（最重要，最持久）
