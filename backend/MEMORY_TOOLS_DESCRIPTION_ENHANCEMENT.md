# 记忆管理工具参数说明增强

## 问题背景

用户指出 `tool_registry.py` 中的 `MEMORY_CONSOLIDATOR_TOOLS` 描述过于简洁，缺少重要的参数说明和使用指导。

## 原始描述（过于简洁）

```python
MEMORY_CONSOLIDATOR_TOOLS = {
    "remember_fact": "添加重要事实到长期记忆。参数：fact(str, 事实内容), category(str, 类别：用户偏好/领域知识/历史结论), priority(int, 优先级1-5)",
    "replace_memory": "替换现有记忆条目。参数：old_text(str, 旧内容), new_text(str, 新内容), category(str, 可选)",
    "remove_memory": "删除过时或错误记忆。参数：text(str, 要删除的内容), category(str, 可选)",
}
```

**问题**：
1. ❌ 没有说明字符限制（3000字符上限）
2. ❌ 没有说明何时使用哪个工具
3. ❌ 没有说明优先级含义（1-5的具体含义）
4. ❌ 没有说明失败场景（记忆已满）
5. ❌ 没有说明删除优先级策略

## 增强后的描述

```python
MEMORY_CONSOLIDATOR_TOOLS = {
    "remember_fact": "添加重要事实到长期记忆。⚠️ **字符限制**：记忆文件上限3000字符，当前使用率会显示在提示词中。当接近上限（>80%）时，优先删除旧内容。参数：fact(str, 要记住的事实，简洁明确，一句话), category(str, 事实类别：用户偏好/领域知识/历史结论/环境信息), priority(int, 优先级1-5，5最高，默认3，用于记忆满时的删除决策)",
    "replace_memory": "替换现有记忆条目。⚠️ **字符限制**：替换操作不会改变总字符数，但可以提高记忆质量。参数：old_text(str, 要替换的旧内容，支持子串匹配), new_text(str, 新的内容), category(str, 可选，用于精确匹配，提高替换准确性)",
    "remove_memory": "删除过时或错误记忆。⚠️ **使用场景**：当记忆文件接近上限（>80%）或已满（100%）时使用。删除优先级：环境信息（临时）> 历史结论（可能过时）> 领域知识（相对稳定）> 用户偏好（最重要）。参数：text(str, 要删除的内容，支持子串匹配), category(str, 可选，用于精确匹配)",
}
```

## 关键增强点

### 1. remember_fact 工具

**增强内容**：
- ✅ **字符限制警告**：明确告知3000字符上限
- ✅ **使用率提醒**：提示词会显示当前使用率
- ✅ **操作建议**：当接近上限（>80%）时，优先删除旧内容
- ✅ **优先级说明**：1-5的含义，5最高，用于记忆满时的删除决策
- ✅ **fact参数说明**：简洁明确，一句话

**示例**：
```
记住：用户喜欢Python编程
- fact: "用户喜欢Python编程"
- category: "用户偏好"
- priority: 5（最高优先级，避免被删除）
```

### 2. replace_memory 工具

**增强内容**：
- ✅ **字符限制说明**：替换操作不会改变总字符数
- ✅ **质量提升**：替换可以提高记忆质量
- ✅ **old_text参数**：支持子串匹配，灵活替换
- ✅ **category参数**：可选，用于精确匹配，提高准确性

**示例**：
```
更新偏好：从"喜欢Java"改为"喜欢Python"
- old_text: "喜欢Java"
- new_text: "喜欢Python"
- category: "用户偏好"（可选，提高匹配准确性）
```

### 3. remove_memory 工具

**增强内容**：
- ✅ **使用场景**：明确告知何时使用（>80%或=100%）
- ✅ **删除优先级**：环境信息 > 历史结论 > 领域知识 > 用户偏好
- ✅ **text参数**：支持子串匹配，灵活删除
- ✅ **category参数**：可选，用于精确匹配

**示例**：
```
删除临时信息：用户今天在公司
- text: "用户今天在公司"
- category: "环境信息"（可选，提高匹配准确性）

删除过时结论：
- text: "2023年数据分析显示..."
- category: "历史结论"（可选）
```

## 参数详细说明

### category 参数（类别）

| 类别 | 含义 | 稳定性 | 优先级 |
|------|------|--------|--------|
| 用户偏好 | 用户习惯、要求、禁忌 | 高 | 最高（5） |
| 领域知识 | 业务规则、概念定义 | 中 | 中（3） |
| 历史结论 | 验证过的规律、结论 | 低 | 低（2） |
| 环境信息 | 临时、会话相关信息 | 极低 | 最低（1） |

### priority 参数（优先级）

| 值 | 含义 | 示例 |
|----|------|------|
| 5 | 最高优先级 | 用户的核心偏好（如"喜欢Python"） |
| 4 | 高优先级 | 重要业务规则 |
| 3 | 中优先级 | 一般性知识（默认） |
| 2 | 低优先级 | 可能过时的结论 |
| 1 | 最低优先级 | 临时环境信息 |

### 使用策略

**场景1：记忆使用率 < 80% (2400字符)**
```
策略：正常添加新记忆
- 优先使用 remember_fact 添加新信息
- 如有重复，使用 replace_memory 更新
```

**场景2：记忆使用率 80-99% (2400-2990字符)**
```
策略：谨慎添加，优先删除
- 优先删除环境信息（priority 1-2）
- 删除过时历史结论（priority 2）
- 删除后，再添加新记忆
```

**场景3：记忆使用率 = 100% (3000字符)**
```
策略：必须先删除
- 工具会返回"MEMORY.md已满"错误
- 必须先使用 remove_memory 删除旧内容
- 删除足够的内容后，再尝试添加
```

## 删除优先级指导

**从高到低**（优先删除）：

1. **环境信息**（priority 1）
   - 临时信息："用户今天在公司"、"用户在使用手机"
   - 会话相关信息："上次对话时间"
   - 理由：极不稳定，下次可能不适用

2. **历史结论**（priority 2）
   - 过时结论："2023年数据分析显示..."
   - 临时结论："初步测试表明..."
   - 理由：可能过时，不如新数据准确

3. **领域知识**（priority 3）
   - 一般性知识："PM2.5是细颗粒物"
   - 相对稳定的概念："React是一个前端框架"
   - 理由：相对稳定，但如果记忆满可以考虑删除

4. **用户偏好**（priority 5）
   - 核心偏好："喜欢Python"、"偏好简洁回答"
   - 个性化要求："不要使用表情符号"
   - 理由：最重要，最持久，应该最后删除

## 实际使用示例

### 示例1：正常添加记忆（使用率50%）

```python
# 当前记忆：1500/3000字符（50%）
# 策略：直接添加

remember_fact(
    fact="用户最近在学习Vue 3",
    category="用户偏好",
    priority=4  # 高优先级，但不是核心偏好
)
```

### 示例2：接近上限，先删除后添加（使用率85%）

```python
# 当前记忆：2560/3000字符（85.3%）
# 策略：先删除环境信息，再添加

# 步骤1：删除临时环境信息
remove_memory(
    text="用户今天在公司",
    category="环境信息"
)

# 步骤2：添加新记忆
remember_fact(
    fact="用户最近在学习Vue 3",
    category="用户偏好",
    priority=4
)
```

### 示例3：记忆已满，必须删除（使用率100%）

```python
# 当前记忆：3000/3000字符（100%）
# 策略：必须先删除，工具会返回错误

# 步骤1：删除多个低优先级记忆
remove_memory(text="用户昨天在公司", category="环境信息")
remove_memory(text="2023年测试结论：...", category="历史结论")

# 步骤2：现在可以添加新记忆
remember_fact(
    fact="用户最近在学习Vue 3",
    category="用户偏好",
    priority=4
)
```
