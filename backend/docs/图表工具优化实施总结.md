## 一、图表工具优化（两层架构）

### 1.1 架构设计

采用**模板库（Template Library） + 智能生成器（Smart Generator）**的两层架构：

- **第一层：模板库** - 针对90%常见场景
  - 快速、可靠、深度优化
  - 保留现有硬编码优化配置
  - 7个预定义模板

- **第二层：智能生成器** - 针对10%新场景
  - LLM驱动的动态图表生成
  - 分析数据特征，自动选择图表类型
  - 灵活、自适应

### 1.2 已实现的文件

#### 图表模板注册表
**文件**: `D:\溯源\backend\app\tools\visualization\generate_chart\chart_templates.py`

**功能**:
- 封装现有visualization.py中的硬编码函数为模板
- 7个预定义模板：
  1. `vocs_analysis` - VOCs分析（3个图表）
  2. `pm_analysis` - 颗粒物分析（2个图表）
  3. `multi_indicator_timeseries` - 多指标时序图（双轴）
  4. `regional_comparison` - 区域对比图
  5. `generic_timeseries` - 通用时序图
  6. `generic_bar` - 通用柱状图
  7. `generic_pie` - 通用饼图

#### 智能图表生成工具
**文件**: `D:\溯源\backend\app\tools\visualization\generate_chart\tool.py`

**功能**:
- 优先尝试使用模板库（scenario参数）
- 模板不存在时，调用LLM分析数据并生成图表
- 支持自定义场景（chart_type_hint、x_field、y_field）
- 失败时提供备用图表配置

**使用示例**:
```python
# 场景1：使用模板（快速）
result = await generate_chart_tool.execute(
    data={"vocs_data": [...], "enterprise_data": [...]},
    scenario="vocs_analysis"
)

# 场景2：智能生成（灵活）
result = await generate_chart_tool.execute(
    data=[{"category": "A", "value": 100}, ...],
    scenario="custom",
    chart_type_hint="bar",
    title="自定义分析"
)
```

## 二、新增工具

### 2.1 组分数据查询工具

**文件**: `D:\溯源\backend\app\tools\query\get_component_data\tool.py`

**功能**:
- 查询VOCs组分数据（用于O3源解析）
- 查询颗粒物组分数据（用于PM2.5/PM10源解析）

**适用范围**: 广东省超级站

**输入参数**:
- `station_name`: 超级站名称
- `component_type`: 组分类型（vocs / particulate）
- `start_time`: 开始时间
- `end_time`: 结束时间

**返回数据**:
```python
{
    "success": True,
    "data": [组分数据列表],
    "count": 数据点数量,
    "summary": "✅ 获取到 X 条VOCs组分数据"
}
```

### 2.2 上风向企业分析工具

**文件**: `D:\溯源\backend\app\tools\analysis\analyze_upwind_enterprises\tool.py`

**功能**:
- 基于风向风速数据，计算上风向扇区
- 在上风向扇区内搜索企业
- 根据距离、行业、排放量等因素评分排序
- 生成高德地图可视化配置

**适用范围**: 广东省

**输入参数**:
- `station_name`: 监测站点名称
- `winds`: 风向风速数据列表 `[{"wd": 度数, "ws": m/s}, ...]`
- `search_range_km`: 搜索半径（默认5.0km）
- `max_enterprises`: 最大企业数量（默认30）
- `top_n`: Top-N企业数量（默认8）

**返回数据**:
```python
{
    "success": True,
    "enterprises": [企业列表],
    "map_url": "高德地图URL",
    "meta": {元数据},
    "summary": "✅ 发现 X 个上风向企业，已生成地图"
}
```

### 2.3 地图生成工具

**文件**: `D:\溯源\backend\app\tools\visualization\generate_map\tool.py`

**功能**:
- 生成用于前端MapPanel组件的高德地图配置
- 展示监测站点位置
- 展示企业标记（带行业、距离等信息）
- 绘制上风向路径（可选）
- 绘制风向扇区（可选）

**输入参数**:
- `station`: 站点信息（包含经纬度、名称）
- `enterprises`: 企业列表
- `upwind_paths`: 上风向路径（可选）
- `sectors`: 风向扇区（可选）

**返回数据**:
```python
{
    "success": True,
    "map_config": {
        "map_center": {"lng": ..., "lat": ...},
        "station": {...},
        "enterprises": [...],
        "upwind_paths": [...],  # 可选
        "sectors": [...]  # 可选
    },
    "summary": "✅ 生成地图配置成功，站点1个，企业X个"
}
```

## 八、技术要点

### 8.1 关键设计决策

1. **两层架构的选择**
   - 平衡了性能（模板）和灵活性（LLM）
   - 符合80/20原则：90%场景用模板，10%场景用LLM

2. **工具元数据的处理**
   - LLMTool基类不接受metadata参数
   - 改为在子类中添加self.metadata属性
   - 不影响工具注册和调用

3. **错误处理策略**
   - 所有工具统一返回格式：`{success, data, summary}`
   - 失败时提供清晰的错误信息和摘要
   - 日志记录详细的上下文信息

### 8.2 技术难点与解决

**难点1**: 硬编码优化 vs 泛化能力
- **解决**: 两层架构，保留优化的同时提供灵活性

**难点2**: LLMTool基类不支持metadata参数
- **解决**: 在子类__init__中添加self.metadata属性

**难点3**: Windows控制台emoji编码问题
- **影响**: 测试脚本显示错误（不影响实际功能）
- **验证**: 通过简化命令验证工具注册成功
