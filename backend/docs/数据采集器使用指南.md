# 数据采集器使用指南

## 概述

数据采集器（Data Fetchers）是后台定时任务，负责从外部API获取数据并存入云数据库。这些数据为LLM工具提供查询支持。

## 已注册的采集器

### 1. ERA5 Fetcher
- **名称**: `era5_fetcher`
- **功能**: 获取ERA5再分析历史气象数据
- **调度**: 每天凌晨2点 (`0 2 * * *`)
- **数据范围**: 昨天的全天数据
- **文件**: `app/fetchers/weather/era5_fetcher.py`

### 2. Observed Weather Fetcher
- **名称**: `observed_fetcher`
- **功能**: 获取气象站观测数据
- **调度**: 每天凌晨3点 (`0 3 * * *`)
- **数据范围**: 昨天的观测数据
- **文件**: `app/fetchers/weather/observed_fetcher.py`

### 3. NASA FIRMS Fetcher
- **名称**: `nasa_firms_fetcher`
- **功能**: 获取火点热点数据
- **调度**: 每小时 (`0 * * * *`)
- **数据范围**: 最近24小时
- **文件**: `app/fetchers/satellite/nasa_firms_fetcher.py`

### 4. CAMS Dust Fetcher
- **名称**: `cams_dust_fetcher`
- **功能**: 获取沙尘AOD（气溶胶光学厚度）数据
- **调度**: 每6小时 (`0 */6 * * *`)
- **数据范围**: 实时预报数据
- **文件**: `app/fetchers/dust/cams_dust_fetcher.py`

## 使用方法

### 方法1：自动启动（推荐）

数据采集器会在FastAPI应用启动时自动启动。

#### 配置要求

在 `.env` 文件中设置：

```bash
# 数据库连接（必需）
DATABASE_URL=postgresql+asyncpg://user:password@host:port/dbname

# 启用自动采集（可选，默认为true）
ENABLE_AUTO_FETCHING=true
```

#### 启动应用

```bash
cd backend
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

日志输出示例：
```
2025-11-02T10:00:00.000000Z [info     ] database_initialized
2025-11-02T10:00:00.100000Z [info     ] fetchers_registered fetchers=['era5_fetcher', 'observed_fetcher', 'nasa_firms_fetcher', 'cams_dust_fetcher']
2025-11-02T10:00:00.200000Z [info     ] fetcher_scheduler_started
2025-11-02T10:00:00.300000Z [info     ] data_fetchers_started
```

### 方法2：禁用自动启动

如果不需要自动采集，在 `.env` 中设置：

```bash
ENABLE_AUTO_FETCHING=false
```

或者删除 `DATABASE_URL` 环境变量。

### 方法3：手动运行单个采集器

通过API端点手动触发：

```bash
# 1. 获取系统状态（查看采集器列表）
curl http://localhost:8000/api/system/status

# 2. 手动运行指定采集器（需要实现API端点）
# 示例：运行ERA5采集器
curl -X POST http://localhost:8000/api/admin/fetcher/run/era5_fetcher
```

### 方法4：使用Python脚本手动运行

创建测试脚本 `run_fetcher_manual.py`：

```python
"""
手动运行数据采集器
"""
import asyncio
from app.fetchers.weather.era5_fetcher import ERA5Fetcher
from app.fetchers.weather.observed_fetcher import ObservedWeatherFetcher
from app.fetchers.satellite.nasa_firms_fetcher import NASAFirmsFetcher
from app.fetchers.dust.cams_dust_fetcher import CAMSDustFetcher
from app.db.database import init_db, close_db
import structlog
import os

# 配置日志
structlog.configure(
    processors=[
        structlog.dev.ConsoleRenderer()
    ],
    logger_factory=structlog.PrintLoggerFactory(),
)

logger = structlog.get_logger()


async def run_single_fetcher(fetcher_name: str):
    """
    运行单个采集器

    Args:
        fetcher_name: 采集器名称
            - era5: ERA5历史数据
            - observed: 观测数据
            - fire: 火点数据
            - dust: 沙尘数据
    """
    # 检查数据库配置
    if not os.getenv("DATABASE_URL"):
        logger.error("DATABASE_URL未配置，无法运行采集器")
        return

    # 初始化数据库
    await init_db()
    logger.info("数据库已初始化")

    try:
        # 选择采集器
        fetcher = None
        if fetcher_name == "era5":
            fetcher = ERA5Fetcher()
        elif fetcher_name == "observed":
            fetcher = ObservedWeatherFetcher()
        elif fetcher_name == "fire":
            fetcher = NASAFirmsFetcher()
        elif fetcher_name == "dust":
            fetcher = CAMSDustFetcher()
        else:
            logger.error(f"未知的采集器: {fetcher_name}")
            return

        logger.info(f"开始运行采集器: {fetcher.name}")

        # 运行采集器
        await fetcher.run()

        logger.info(f"采集器运行完成: {fetcher.name}")

    except Exception as e:
        logger.error(f"采集器运行失败", error=str(e), exc_info=True)

    finally:
        # 关闭数据库连接
        await close_db()
        logger.info("数据库连接已关闭")


async def run_all_fetchers():
    """运行所有采集器（用于批量回填历史数据）"""
    if not os.getenv("DATABASE_URL"):
        logger.error("DATABASE_URL未配置")
        return

    await init_db()

    fetchers = [
        ERA5Fetcher(),
        ObservedWeatherFetcher(),
        NASAFirmsFetcher(),
        CAMSDustFetcher(),
    ]

    for fetcher in fetchers:
        try:
            logger.info(f"运行采集器: {fetcher.name}")
            await fetcher.run()
            logger.info(f"完成: {fetcher.name}")
        except Exception as e:
            logger.error(f"失败: {fetcher.name}", error=str(e))

    await close_db()


if __name__ == "__main__":
    import sys

    if len(sys.argv) < 2:
        print("用法:")
        print("  python run_fetcher_manual.py <fetcher_name>")
        print("  python run_fetcher_manual.py all")
        print()
        print("可用的采集器:")
        print("  era5      - ERA5历史气象数据")
        print("  observed  - 观测站气象数据")
        print("  fire      - NASA FIRMS火点数据")
        print("  dust      - CAMS沙尘AOD数据")
        print("  all       - 运行所有采集器")
        sys.exit(1)

    fetcher_name = sys.argv[1]

    if fetcher_name == "all":
        asyncio.run(run_all_fetchers())
    else:
        asyncio.run(run_single_fetcher(fetcher_name))
```

#### 使用示例

```bash

python run_fetcher_manual.py era5

# 运行观测数据采集器
python run_fetcher_manual.py observed

# 运行所有采集器
python run_fetcher_manual.py all
```

## 数据回填（Backfill）

如果需要回填历史数据，有两种方式：

### 方式1：修改采集器代码

在采集器的 `fetch_and_store()` 方法中修改日期范围：

```python
# 原代码（获取昨天）
yesterday = (datetime.now() - timedelta(days=1)).strftime("%Y-%m-%d")

# 修改为回填最近30天
for days_ago in range(1, 31):
    target_date = (datetime.now() - timedelta(days=days_ago)).strftime("%Y-%m-%d")
    # ... 执行采集逻辑
```

### 方式2：创建专门的回填脚本

```python
"""
历史数据回填脚本
"""
import asyncio
from datetime import datetime, timedelta
from app.fetchers.weather.era5_fetcher import ERA5Fetcher
from app.db.database import init_db, close_db
import structlog

logger = structlog.get_logger()


async def backfill_era5_data(start_date: str, end_date: str):
    """
    回填ERA5历史数据

    Args:
        start_date: 开始日期，格式 YYYY-MM-DD
        end_date: 结束日期，格式 YYYY-MM-DD
    """
    await init_db()

    fetcher = ERA5Fetcher()

    start = datetime.strptime(start_date, "%Y-%m-%d")
    end = datetime.strptime(end_date, "%Y-%m-%d")

    current_date = start
    while current_date <= end:
        date_str = current_date.strftime("%Y-%m-%d")
        logger.info(f"回填日期: {date_str}")

        # 这里需要修改fetcher的内部日期逻辑
        # 或者直接调用底层方法

        current_date += timedelta(days=1)
        await asyncio.sleep(1)  # 避免API限流

    await close_db()


if __name__ == "__main__":
    # 回填最近30天的数据
    start = (datetime.now() - timedelta(days=30)).strftime("%Y-%m-%d")
    end = (datetime.now() - timedelta(days=1)).strftime("%Y-%m-%d")

    asyncio.run(backfill_era5_data(start, end))
```

## 常见问题

### Q1: 采集器没有自动运行？

**检查清单**:
1. 确认 `DATABASE_URL` 已配置
2. 确认 `ENABLE_AUTO_FETCHING=true`
3. 检查日志是否有 `fetcher_scheduler_started`
4. 确认调度时间是否正确（使用cron表达式）

### Q2: 采集器运行失败？

**可能原因**:
1. 外部API不可访问（网络问题）
2. API密钥未配置或已过期
3. 数据库连接失败
4. 数据格式变化导致解析错误

**解决方法**:
- 查看详细日志 (`structlog` 会输出错误堆栈)
- 测试外部API连接性
- 验证数据库连接
- 检查API响应格式

### Q3: 如何查看已存储的数据？

```bash
# 通过气象数据查询API
curl -X POST http://localhost:8000/api/weather/query \
  -H "Content-Type: application/json" \
  -d '{
    "data_type": "era5",
    "lat": 23.13,
    "lon": 113.26,
    "start_time": "2025-11-01T00:00:00",
    "end_time": "2025-11-01T23:59:59"
  }'

# 或通过数据统计API
curl http://localhost:8000/api/weather/stats
```

### Q4: 数据采集频率可以修改吗？

可以。修改采集器的 `schedule` 参数（cron表达式）：

```python
class ERA5Fetcher(DataFetcher):
    def __init__(self):
        super().__init__(
            name="era5_fetcher",
            schedule="0 2 * * *",  # 修改这里
            ...
        )
```

常用cron表达式：
- `0 * * * *` - 每小时
- `*/30 * * * *` - 每30分钟
- `0 */6 * * *` - 每6小时
- `0 2 * * *` - 每天2点
- `0 2 * * 0` - 每周日2点

### Q5: 如何暂停/恢复采集器？

```python
from app.services.lifecycle_manager import get_fetcher_scheduler

scheduler = get_fetcher_scheduler()

# 暂停
scheduler.pause("era5_fetcher")

# 恢复
scheduler.resume("era5_fetcher")

# 停止所有
scheduler.stop()

# 启动所有
scheduler.start()
```

## 监控和维护

### 建议的监控指标

1. **采集成功率**: `success_count / total_count`
2. **采集延迟**: 从调度时间到完成的时长
3. **数据量增长**: 每日新增记录数
4. **失败日志**: 错误类型和频率

### 日志位置

- 应用日志: 标准输出（stdout）
- 结构化日志: JSON格式（生产环境）
- 开发日志: 彩色格式（开发环境）

### 数据清理策略

建议定期清理旧数据：

```sql
-- 删除90天前的ERA5数据
DELETE FROM weather_data_era5 WHERE time < NOW() - INTERVAL '90 days';

-- 删除30天前的火点数据
DELETE FROM fire_hotspots WHERE acq_date < NOW() - INTERVAL '30 days';
```

## 扩展开发

### 创建新的采集器

1. 继承 `DataFetcher` 基类：

```python
from app.fetchers.base.fetcher_interface import DataFetcher

class MyCustomFetcher(DataFetcher):
    def __init__(self):
        super().__init__(
            name="my_custom_fetcher",
            description="My custom data fetcher",
            schedule="0 * * * *",  # 每小时
            version="1.0.0"
        )

    async def fetch_and_store(self):
        """实现数据获取和存储逻辑"""
        # 1. 调用外部API
        # 2. 数据清洗和转换
        # 3. 存入数据库
        pass
```

2. 在 `lifecycle_manager.py` 中注册：

```python
from app.fetchers.my_custom_fetcher import MyCustomFetcher

def initialize_fetchers():
    # ... 现有代码
    fetcher_scheduler.register(MyCustomFetcher())
```
