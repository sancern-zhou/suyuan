# ReadFile 工具 - Word XML 智能分层功能（无意图检测版本）

## ✅ 实现完成

已为 `read_file` 工具添加 Word XML 智能分层读取功能。

## 🎯 核心原则

**明确说明**：系统**不进行意图检测**，完全由 LLM 通过参数控制模式。

### ❌ 不做什么
- ❌ 不分析用户问题内容（如"编辑"、"修改"）
- ❌ 不提取关键词推断意图
- ❌ 不进行自动意图推断

### ✅ 做什么
- ✅ 根据文件大小自动选择默认模式
- ✅ 完全由 LLM 通过参数显式控制
- ✅ 提供三种模式供 LLM 选择

## 📊 三种模式

| 模式 | 触发条件 | Token 消耗 | 格式保留 | LLM 使用场景 |
|------|----------|------------|----------|--------------|
| **text** | 文件 ≥ 100KB | ~10% | 无 | 快速阅读大文档 |
| **structured** | 文件 < 100KB | ~20% | 部分（标题、表格） | 理解文档结构 |
| **raw** | `raw_mode=True` | 100% | 完整 | 精确编辑格式 |

## 💡 LLM 使用场景

### 场景 1：默认行为（推荐）

```
用户：查看这个文档

LLM 调用：
read_file("document.xml")

系统行为：
1. 检测文件大小：80KB
2. 自动选择：structured 模式（因为 < 100KB）
3. 返回：结构化摘要（标题、表格、段落）
```

### 场景 2：LLM 明确需要编辑

```
用户：修改标题样式为加粗

LLM 调用：
read_file("document.xml", raw_mode=True)

系统行为：
1. 检测到 raw_mode=True
2. 返回：完整原始 XML
3. LLM 使用 edit_file 修改格式
```

### 场景 3：控制读取量

```
用户：只读取前 50 个段落

LLM 调用：
read_file("document.xml", max_paragraphs=50)

系统行为：
1. 应用段落数限制
2. 返回：前 50 个段落的优化内容
```

## 🎨 设计理念

符合项目核心哲学：

> **"LLM 完全自由 + 系统智能适配"**
> **"LLM 显式决策 > 系统自动推断"**

- **LLM 完全自由**：通过参数自由控制模式
- **系统智能适配**：基于文件大小自动选择合理的默认模式
- **不进行意图推断**：避免关键词误判，保持可预测性

## ⚠️ 重要说明

### 系统不会做的事情

1. ❌ **不分析用户问题内容**
   - 不检测"编辑"、"修改"、"查看"等关键词
   - 不进行自然语言意图分析
   - 不猜测用户想要什么

2. ❌ **不自动切换模式**
   - 不会因为用户说"编辑"就自动用 raw 模式
   - 不会因为用户说"查看"就自动用 text 模式

3. ❌ **不进行上下文推断**
   - 不分析对话历史
   - 不考虑之前的操作

### 系统会做的事情

1. ✅ **根据文件大小选择默认模式**
   - 小文件（< 100KB）→ structured 模式
   - 大文件（≥ 100KB）→ text 模式

2. ✅ **完全遵守 LLM 的参数**
   - `raw_mode=True` → 总是返回 raw 模式
   - `include_formatting=True` → 总是返回 structured 模式
   - `max_paragraphs=N` → 总是限制段落数

3. ✅ **提供清晰的模式说明**
   - 在工具描述中明确说明每种模式的用途
   - 在示例中展示如何使用参数

## 📝 关键区别

### 与 LobsterAI 的区别

| 项目 | 模式控制方式 | 意图检测 | LLM 自由度 |
|------|-------------|----------|-------------|
| **LobsterAI** | 工作流树（提示词指导） | 无 | 中（需遵循规则） |
| **本项目** | LLM 参数控制 | 无 | 高（完全自由） |

### 与关键词检测的区别

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| **关键词检测** | 自动化 | 误判率高、不可预测 | ❌ 不采用 |
| **LLM 参数控制** | 精确、可预测 | LLM 需要了解参数 | ✅ **采用** |

## 🚀 使用建议

### 对于 LLM

1. **默认场景**：直接调用 `read_file("document.xml")`
   - 系统会根据文件大小自动选择合理的默认模式

2. **需要编辑**：显式设置 `raw_mode=True`
   - 明确告诉系统你需要完整 XML

3. **控制量**：使用 `max_paragraphs=N`
   - 进一步控制 token 消耗

### 对于用户

1. **简单阅读**：直接说"查看文档"
   - LLM 会自动使用默认模式

2. **精确编辑**：明确说"获取完整 XML 用于编辑"
   - LLM 会设置 `raw_mode=True`

3. **快速浏览**：说"只读取前 50 段"
   - LLM 会设置 `max_paragraphs=50`

## 🎓 总结

这个实现的核心思想是：

> **"LLM 比系统更懂自己的意图"**
> **"参数控制 > 意图推断"**

- ✅ LLM 通过参数精确控制行为
- ✅ 系统只负责根据文件大小选择合理的默认值
- ✅ 避免关键词误判导致不可预测的结果
- ✅ 保持简单、可预测、高效
