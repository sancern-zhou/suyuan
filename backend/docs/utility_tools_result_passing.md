# Utility 工具完整结果传递分析

## 工具分类

### 需要完整结果传递的工具

**1. grep（搜索文件内容）**
- 返回：匹配的代码行、文件路径、行号
- 需要完整传递：✅ 是
- 原因：LLM 需要看到所有匹配结果才能理解代码结构

**2. glob/search_files（搜索文件名）**
- 返回：文件路径列表
- 需要完整传递：✅ 是
- 原因：LLM 需要看到所有匹配的文件才能选择正确的文件

**3. list_directory（列出目录）**
- 返回：文件和目录列表、元信息
- 需要完整传递：✅ 是
- 原因：LLM 需要看到完整目录结构才能导航

**4. read_file（读取文件）**
- 返回：文件内容（文本/图片分析/PDF）
- 需要完整传递：✅ 已实现
- 状态：已在 loop.py 中识别

### 不需要完整结果传递的工具

**5. edit_file（编辑文件）**
- 返回：编辑成功/失败、修改统计
- 需要完整传递：❌ 否
- 原因：只需要知道是否成功，不需要看完整文件内容

**6. write_file（写入文件）**
- 返回：写入成功/失败、文件大小
- 需要完整传递：❌ 否
- 原因：只需要知道是否成功，不需要看完整文件内容

## 结论

需要在 `loop.py` 中添加识别的工具：
- ✅ grep
- ✅ glob (search_files)
- ✅ list_directory

不需要特殊处理的工具：
- ❌ edit_file（只返回摘要即可）
- ❌ write_file（只返回摘要即可）
