# 复杂查询工具策略手册

此手册只注入 `complex_query_planner` 的内部 LLM 上下文，不注入主 Agent 常驻上下文。目标是让 planner 在复杂空气质量查询中选择正确工具、字段路径和执行策略。

## 总原则

1. 优先选择能一次返回目标统计结果的专用工具，避免多次查询后手算。
2. 用户问统计报表、综合指数、达标率、超标天数、首要污染物比例时，优先使用 `query_new_standard_report` 或 `query_old_standard_report`。
3. 用户问同比、环比、变化、下降、上升、改善、恶化时，优先使用对比工具 `compare_standard_reports` 或 `compare_old_standard_reports`。
4. 用户问新旧标准影响、差异、同一时期新旧口径对比时，使用 `query_standard_comparison`。
5. 用户问 168 城市全国排名、排名变化或全国发布数据时，优先用 `execute_sql_query` 查询预计算统计表，不要使用广东省内查询工具代替全国排名。
6. 用户问小时级、站点级或区域对比时，选择对应的小时、站点或区域工具，不要用城市日报工具硬凑。
7. 缺少时间范围、城市范围或标准口径时，计划中返回 `error` 和 `warnings`，不要编造参数。

## result vs data_id

- `result` 是专用工具已经计算好的统计结果，报告生成、趋势分析和问答解释应优先读取它。
- `data_id` 是完整明细数据的存储标识，通常用于后续聚合、图表或查看逐日/逐小时记录。
- 不要从 `data_id` 读取统计汇总字段，例如 `exceed_days_by_pollutant`、`primary_pollutant_exceed_days`、`composite_index` 等，除非对应工具明确说明这些字段在明细中存在。
- 对比工具的 `result["城市名"]["query_period"]` 已包含查询期完整统计，`comparison_period` 已包含对比期完整统计，不要再额外调用单时段统计工具查询同一时间段。

## 统计报表策略

- 新标准统计：`query_new_standard_report`
- 旧标准统计：`query_old_standard_report`
- 新旧标准同周期对比：`query_standard_comparison`
- 新标准不同时段对比：`compare_standard_reports`
- 旧标准不同时段对比：`compare_old_standard_reports`
- 站点新标准统计：`query_station_new_standard_report`

多城市查询时，报表工具可能返回各城市结果和 `province_wide`。如用户问“广东省整体/全省汇总”，优先读取 `province_wide`；如用户问各城市排名或对比，读取各城市节点。

## 日数据、小时数据与聚合

- 用户明确要逐日数据、日期列表、日变化曲线时，用 `query_gd_suncere_city_day_new` 或旧标准日数据工具。
- 用户要小时变化、小时峰值或日内过程时，用 `query_gd_suncere_city_hour` 或 `query_gd_suncere_station_hour_new`。
- 用户要月度分组统计但专用报表只返回累计区间结果时，按月份拆分为多个报表调用，或查询日数据后交给聚合工具；不要把一个累计结果误当成月序列。

## SQL 查询策略

`execute_sql_query` 适合：

- 168 城市全国排名、全国发布数据、预计算排名字段。
- 需要直接访问统计表中的字段。
- 需要 CTE、窗口函数、JOIN 或复杂筛选，并且专用工具不能覆盖。

SQL 约束：

- 使用 SQL Server 语法。
- 中文字符串加 `N` 前缀，例如 `N'广州'`。
- 表结构不确定时，计划中应先调用表结构查询或说明需要 `describe_table`/数据字典，而不是猜字段。
- 已预计算的排名字段优先直接查询，不要重复用窗口函数计算。

## 排名策略

- 全国 168 城市排名：优先查 `city_168_statistics_new_standard` 或相关全国发布表。
- 广东省内城市相互排序：如果目标指标是专用报表字段，可查询报表后按 `result` 排序；如果数据库已有预计算排名字段，优先 SQL。
- 相同指标值必须提示主 Agent 使用并列排名。

## 同比/环比策略

- 空气质量统计指标同比/环比：用对应标准的 compare 工具。
- 用户没有指定对比期时，根据当前时间推断常见口径，并在计划 reasoning 中写清楚。例如“今年以来同比”对应今年以来 vs 去年同期。
- 排名变化同比/环比：如果排名来自全国统计表，使用 SQL 取两个时间点的排名字段。

## 新旧标准策略

- 用户只说“新标准”或没有特殊说明时，优先 HJ 633-2026 新标准工具。
- 用户明确“旧标准”“2013标准”时，用旧标准工具。
- 用户问“新标准影响”“新旧标准差异”“标准切换前后口径差异”时，用 `query_standard_comparison`。

## 站点与区域策略

- 用户给出站点名称、站点编码或监测点位时，优先站点工具。
- 用户问区域、省内片区、城市组之间对比时，用区域对比工具或拆分城市组调用报表工具。
- 不要用城市级结果回答站点级问题，除非用户允许汇总到城市级。

## 常见误用

- 错误：为了统计超标天数，先查日数据再让主 Agent 手算。正确：优先报表工具。
- 错误：对比两个时期时分别调用两次单时段报表再手算。正确：使用 compare 工具。
- 错误：全国排名使用广东省内查询工具。正确：使用全国统计表或全国空气质量工具。
- 错误：重复查询全省和部分城市。正确：如果已查询更大城市集合，从结果中提取子集。
- 错误：只返回调用计划，不告诉主 Agent 读取哪个字段。正确：在 `result_usage` 和 `field_paths` 中写清楚。
