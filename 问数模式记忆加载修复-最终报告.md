# 问数模式记忆加载修复 - 最终报告

**修复日期**: 2026-04-06
**状态**: ✅ 已验证成功

---

## 问题描述

问数模式（query模式）在未登录状态下无法自动加载长期记忆，导致记忆增强功能失效。

### 初始症状

- 用户问数时，系统无法从 `MEMORY.md` 加载记忆
- 日志显示 `longterm_memory_disabled=True`
- 日志显示 `context_length=77`（只有空模板）

---

## 问题分析

### 第一层问题：路径计算错误

**文件**: `backend/app/agent/memory/memory_store.py:109-111`

**最初尝试**（失败）：
```python
# 全局目录：直接使用模式目录（backend_data_registry/memory/{mode}/）
# 例如：backend_data_registry/memory/query/MEMORY.md
return mode_dir
```

**问题**：
- 新路径：`backend_data_registry/memory/query/MEMORY.md`（空文件）
- 旧路径：`backend_data_registry/memory/query/query_global_shared/MEMORY.md`（有完整数据）
- 导致读取了空文件，记忆增强失败

### 第二层问题：loop.py 硬编码禁用

**文件**: `backend/app/agent/core/loop.py:236-241`

**原代码**：
```python
# Step 1: 使用原始查询（长期记忆增强已移除）
enhanced_query = user_query
logger.info(
    "query_processing",
    query_length=len(user_query),
    longterm_memory_disabled=True  # ❌ 硬编码为True
)
```

**问题**：
- 硬编码 `longterm_memory_disabled=True`
- 即使 `react_agent.py` 加载了记忆，这里也忽略了

---

## 解决方案

### 修改 1：恢复旧路径（向后兼容）

**文件**: `backend/app/agent/memory/memory_store.py:109-111`

```python
# 全局目录：使用 {mode}_global_shared 目录（向后兼容已有记忆文件）
global_dir = mode_dir / f"{mode}_global_shared"
global_dir.mkdir(parents=True, exist_ok=True)
return global_dir
```

**理由**：
- 保持与已有记忆文件的兼容性
- 避免数据迁移风险
- 最小化改动

### 修改 2：优化记忆增强检测

**文件**: `backend/app/agent/core/loop.py:236-252`

```python
# Step 1: 使用增强查询（已在 react_agent.py 中加载记忆）
# user_query 参数已经是增强后的查询（包含记忆上下文）
enhanced_query = user_query

# 检测是否有记忆增强（通过比较原始查询和增强查询）
has_memory = (
    original_query is not None and
    len(user_query) > len(original_query) and
    "长期记忆" in user_query
)

logger.info(
    "query_processing",
    query_length=len(user_query),
    original_length=len(original_query) if original_query else 0,
    has_memory_enhancement=has_memory,
    longterm_memory_disabled=not has_memory
)
```

**改进**：
- 动态检测记忆增强（比较查询长度和关键词）
- 不再硬编码 `longterm_memory_disabled=True`
- 更准确的日志记录
