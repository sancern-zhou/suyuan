# 快速溯源工作流程对比分析

## 对比时间
2026-03-29

## 对比范围

- **原系统**: `QuickTraceExecutor` (backend/app/agent/executors/quick_trace_executor.py)
- **新系统**: `quick_trace_standard_multi_agent.md` (backend/config/task_lists/)

---

## 一、原工作流程分析

### 原系统工具链（QuickTraceExecutor）

```
1. 历史气象数据（ERA5）
   - 工具：get_weather_data
   - 时间范围：前3天
   - 目的：分析历史气象条件

2. 天气预报
   - 工具：get_weather_forecast
   - 时间范围：15天预报 + 昨天+今天00:00~当前时刻
   - 目的：获取完整气象数据（包含边界层高度）

3. 区域对比
   - 工具：_get_air_quality_from_db
   - 内容：
     * 周边8城市历史12小时数据
     * 未来7天日预报数据
   - 目的：判断本地生成 vs 区域传输

4. 轨迹分析
   - 工具：meteorological_trajectory_analysis
   - 内容：后向轨迹分析
   - 超时时间：180秒
   - 目的：追溯污染来源

5. 天气形势图解读
   - 工具：get_weather_situation_map
   - 内容：中央气象台天气形势图AI解读（通义千问VL）
   - 目的：天气形势分析
```

### 执行特点

- **并行执行**: 所有工具完全独立，并行执行
- **超时处理**: 轨迹分析超时则跳过，继续生成报告
- **固定城市**: 目前仅支持济宁
- **总耗时**: 3-5分钟（轨迹分析超时则2-3分钟）

## 二、新工作流程分析

### 新系统任务清单（quick_trace_standard_multi_agent.md）

```
任务1：定位站点
  工具：get_nearby_stations
  输出：站点信息（经纬度、站点ID）

任务2：气象数据分析【气象专家子Agent】
  专家：气象专家
  工具：call_sub_agent → get_weather_data
  依赖：任务1
  输出：气象条件分析报告（MD格式）

任务3：后向轨迹分析【轨迹专家子Agent】
  专家：轨迹专家
  工具：call_sub_agent → meteorological_trajectory_analysis
  依赖：任务2
  输出：后向轨迹分析报告（MD格式）+ 轨迹图

任务4：上风向企业分析【轨迹专家子Agent】
  专家：轨迹专家
  工具：call_sub_agent → analyze_upwind_enterprises
  依赖：任务3
  可选：是（如无企业数据可跳过）
  输出：上风向企业分析报告（MD格式）

任务5：污染物组分分析【化学专家子Agent】
  专家：化学专家
  工具：call_sub_agent → get_vocs_data/get_pm25_ionic
  依赖：任务1（可与任务2-4并行）
  可选：是（如无数据可跳过）
  输出：污染物组分分析报告（MD格式）

任务6：生成综合报告【综合报告专家子Agent】
  专家：报告专家
  工具：call_sub_agent
  依赖：所有任务完成
  输出：完整的综合溯源报告（MD格式）
```

### 执行特点

- **顺序执行**: 大部分任务需要按顺序执行（任务1→2→3→4，任务6）
- **并行机会**: 任务5可与任务2-4并行
- **通用站点**: 支持任意站点（通过get_nearby_stations）
- **预计耗时**: 约3分钟

## 四、关键问题分析

### 🔴 严重问题：缺少区域对比分析

**原系统价值链**：
```
周边城市数据 → 时序对比 → 判断本地生成 vs 区域传输 → 溯源结论
```

**新系统缺失**：
- ❌ 没有周边城市数据获取
- ❌ 没有时序对比分析
- ❌ 无法判断本地生成 vs 区域传输
- ❌ 溯源结论缺乏关键依据

**影响**：
- 溯源结论的准确性大幅下降
- 无法区分本地排放和区域传输
- 管控建议可能失焦

**解决方案**：
```python
# 新增任务：区域对比分析
任务2.5：区域对比分析【气象专家子Agent】
- 专家：气象专家
- 工具：call_sub_agent
- 内容：
  1. 获取周边站点空气质量数据
  2. 时序对比分析
  3. 判断本地生成 vs 区域传输
- 依赖：任务1完成
- 输出：区域对比分析报告（MD格式）
```

### 🟡 中等问题：缺少天气预报数据

**原系统**：
- 15天天气预报
- 昨天+今天00:00~当前时刻完整数据（包含边界层高度）

**新系统**：
- 只有历史气象数据
- 缺少未来趋势判断

**影响**：
- 无法判断未来污染趋势
- 缺少改善时机预测
- 管控建议缺少前瞻性

**解决方案**：
在任务2（气象数据分析）中增加天气预报工具调用：
```python
# 气象专家应该调用：
1. get_weather_data（历史数据）
2. get_weather_forecast（15天预报）
```

### 🟡 中等问题：缺少天气形势图解读

**原系统**：
- 中央气象台天气形势图AI解读（通义千问VL）

**新系统**：
- 缺少此功能

**影响**：
- 缺少大尺度天气形势分析
- 无法判断高压脊、低压槽等天气系统
- 污染扩散条件分析不够全面

**解决方案**：
```python
# 可选任务：天气形势图解读
任务2.6：天气形势图解读【气象专家子Agent】
- 专家：气象专家
- 工具：call_sub_agent → get_weather_situation_map
- 可选：是
- 输出：天气形势图解读报告（MD格式）
```

### 🟢 小问题：化学专家提示词路径不一致

**问题**：
- 任务清单中写的是：`chemical_expert.md`
- 实际文件是：`chemical_expert_pm.md` 和 `chemical_expert_o3.md`

**解决方案**：
根据污染物类型动态选择提示词文件：
```python
expert_prompt_file = (
    "backend/config/prompts/chemical_expert_pm.md"
    if pollutant == "PM2.5"
    else "backend/config/prompts/chemical_expert_o3.md"
)
```

## 五、改进建议

### 优先级1（必须）：补充区域对比分析

**原因**：区域对比是判断本地生成 vs 区域传输的关键依据，缺失会严重影响溯源结论的准确性。

**实施方案**：
1. 在任务清单中新增"区域对比分析"任务
2. 气象专家提示词已包含"区域时序对比分析"框架
3. 需要确认是否有周边站点数据查询工具

### 优先级2（重要）：补充天气预报数据

**原因**：天气预报是判断未来污染趋势和改善时机的关键。

**实施方案**：
1. 在任务2（气象数据分析）中增加get_weather_forecast工具调用
2. 气象专家提示词已包含"气象预报与污染潜势"框架

### 优先级3（建议）：补充天气形势图解读

**原因**：天气形势图解读可以提供大尺度天气系统分析，提升分析深度。

**实施方案**：
1. 添加可选的天气形势图解读任务
2. 作为气象专家的增强功能

### 优先级4（优化）：修正化学专家提示词路径

**原因**：确保任务清单与实际文件路径一致。

**实施方案**：
1. 根据污染物类型动态选择提示词文件
2. 在任务清单中明确说明

## 六、总结

### ✅ 新系统的优势

1. **通用性强**: 支持任意站点，不限于固定城市
2. **专业性强**: 每个专家有独立的专业提示词
3. **功能丰富**: 新增上风向企业分析、污染物组分分析
4. **用户可见**: 阶段性报告生成，用户可以看到分析进度

### ❌ 新系统的主要问题

1. **缺少区域对比**: 无法判断本地生成 vs 区域传输（严重）
2. **缺少天气预报**: 无法判断未来污染趋势（重要）
3. **缺少天气形势图**: 缺少大尺度天气系统分析（建议）
4. **执行效率**: 顺序执行可能导致总耗时增加

### 🎯 核心建议

**最低可行方案**：
- ✅ 补充区域对比分析（必须）
- ✅ 补充天气预报数据（重要）
- ✅ 修正化学专家提示词路径（必须）

**完整方案**：
- 以上最低可行方案 +
- ✅ 补充天气形势图解读（可选）
- ✅ 优化并行执行策略（可选）
