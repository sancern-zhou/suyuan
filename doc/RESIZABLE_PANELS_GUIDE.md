# 可拖动分隔条功能 - 使用指南

## 新增功能

### ✨ 图表与文字区域可调整宽度

每个分析模块（气象分析、区域对比、组分分析）的图表和文字区域之间添加了**可拖动的分隔条**，用户可以根据需要调整左右两侧的宽度比例。

## 功能特性

### 1. 可拖动分隔条
- **位置**: 图表区域和文字区域之间
- **外观**: 灰色竖线，鼠标悬停时变为蓝色
- **拖动**: 鼠标左键按住分隔条，左右拖动即可调整宽度

### 2. 宽度限制
- **左侧图表区域**: 最小 40%，最大 75%
- **右侧文字区域**: 最小 25%，最大 60%
- **默认比例**: 图表 65% : 文字 35%

### 3. 文字内容完整显示
- ✅ 移除了之前的 520px 高度限制
- ✅ 文字区域自动适应内容高度
- ✅ 内容过长时自动出现滚动条

## 使用方法

### 1. 调整宽度比例

```
┌─────────────────────────────────────────────┐
│          气象与上风向分析                    │
├────────────────────┬──│──┬─────────────────┤
│ 图表区域           │  │  │  文字区域       │
│                    │  ││  │                 │
│ - O3趋势图         │  拖│  │  分析内容...   │
│ - 企业分布地图     │  动│  │                 │
│                    │  条│  │                 │
└────────────────────┴──│──┴─────────────────┘
                        ↑
                   鼠标按住拖动
```

**操作步骤**:
1. 将鼠标移动到图表和文字之间的分隔线上
2. 鼠标光标变为 `↔` 双向箭头
3. 按住鼠标左键不放
4. 左右拖动调整宽度
5. 松开鼠标完成调整

### 2. 视觉反馈

**鼠标悬停**:
- 分隔条从灰色变为蓝色
- 鼠标光标变为 `col-resize` (↔)

**拖动中**:
- 分隔条保持蓝色高亮
- 页面禁用文字选择（避免误操作）
- 鼠标移动到页面任何位置都会响应

**拖动完成**:
- 松开鼠标后，分隔条恢复灰色
- 恢复正常的文字选择功能

## 故障排查

### 问题 1: 分隔条无法拖动

**检查**:
- 浏览器控制台是否有 JavaScript 错误
- ResizablePanels 组件是否正确导入
- 鼠标光标是否变为 `↔`

**解决**:
- 刷新页面 (Ctrl+F5) 清除缓存
- 检查 ModuleCard.tsx 是否正确使用 ResizablePanels

### 问题 2: 文字内容仍然显示不全

**检查**:
- TextPanel 是否有 `height: '100%'` 样式
- 父容器是否有高度限制
- 是否出现滚动条

**解决**:
- 检查 TextPanel.tsx line 38: `height: '100%'`
- 调整分隔条给文字区域更多空间

### 问题 3: 拖动时页面卡顿

**原因**:
- 频繁的 DOM 更新导致性能问题
- 浏览器渲染压力大

**优化**:
- 使用 `requestAnimationFrame` 节流
- 减少不必要的状态更新

## 设计理念

### 为什么需要可拖动分隔条？

1. **灵活性**: 不同用户对图表和文字的关注度不同
2. **适应性**: 不同内容长度需要不同的空间分配
3. **用户控制**: 让用户自主决定信息展示方式

### 为什么选择这些默认值？

- **65% 图表 vs 35% 文字**: 视觉信息通常需要更多空间
- **最小 40% 图表**: 确保图表不会太小而难以阅读
- **最小 25% 文字**: 确保文字区域至少可显示完整段落
