# 历史对话上下文修复验证指南

## 修复内容

### 1. 核心修复：`react_agent.py` 的 `_get_or_create_session()` 方法

**修改位置**：`backend/app/agent/react_agent.py:406-454`

**修复内容**：
- 在创建新会话前，先尝试从 `SessionManager` 恢复已保存的会话
- 如果恢复成功，立即加载历史消息到 `memory_manager.session`
- 返回 `created_new=False` 表示这是恢复的会话，不是新建的

**关键代码**：
```python
# ✅ 新增：尝试从 SessionManager 恢复会话
if session_id and not reset_session:
    try:
        from app.agent.session import get_session_manager
        session_manager = get_session_manager()
        saved_session = session_manager.load_session(session_id)

        if saved_session and saved_session.conversation_history:
            # 创建 memory_manager 并加载历史消息
            memory_manager = HybridMemoryManager(...)
            memory_manager.session.load_history_messages(saved_session.conversation_history)
            # ...
            return session_id, memory_manager, False  # False 表示不是新建
    except Exception as e:
        logger.warning("react_session_restore_failed", ...)
```

## 常见问题排查

### 问题1：会话恢复失败，日志显示 "react_session_restore_failed"

**可能原因**：
- 会话文件损坏
- 会话文件不存在
- 消息格式不匹配

**排查方法**：
```bash
# 检查会话文件
cat backend_data_registry/sessions/{session_id}.json | jq '.'

# 检查 conversation_history 格式
cat backend_data_registry/sessions/{session_id}.json | jq '.conversation_history[0]'
```

**解决方法**：
- 如果会话文件损坏，删除该会话，重新创建
- 如果消息格式不匹配，检查前端保存格式是否正确

### 问题2：历史消息加载后，LLM仍然无法访问

**可能原因**：
- 上下文压缩过度，历史消息被压缩丢失
- LLM 模型不支持长上下文
- 消息格式转换错误

**排查方法**：
查看日志中的上下文构建信息：
```
get_messages_for_llm_success  session_id=xxx total_history_length=N
context_builder_result  has_conversation_history=true
```

**解决方法**：
- 增加 `max_context_tokens` 参数
- 检查消息格式转换逻辑
- 考虑使用支持长上下文的 LLM 模型

### 问题3：日志显示 "session_already_has_history" 但历史长度为0

**可能原因**：
- 会话恢复时，`conversation_history` 为空
- 消息加载失败

**排查方法**：
查看日志中的消息加载统计：
```
history_messages_loaded  total_input=N successfully_loaded=0 skipped=N errors=0
```

**解决方法**：
- 检查消息格式是否符合预期
- 检查是否有加载错误
- 参考 `session_memory.py` 中的日志定位具体问题

## 性能影响评估

### 内存占用

- **修复前**：每次恢复会话都创建新的 `HybridMemoryManager`，历史消息无法复用
- **修复后**：从 `SessionManager` 恢复会话，复用已保存的历史消息
- **影响**：减少内存占用，提高会话恢复速度

### 响应时间

- **修复前**：恢复会话后，Agent 需要重新处理历史消息
- **修复后**：历史消息已加载到 `memory_manager.session`，Agent 可以直接使用
- **影响**：减少首次响应时间

### 磁盘I/O

- **修复前**：每次都从磁盘加载会话文件
- **修复后**：首次加载后，会话保存在 `_session_store` 内存缓存中
- **影响**：减少磁盘I/O，提高性能
