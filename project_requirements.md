# 大气污染溯源分析系统需求说明书

## 1. 项目概述

**项目名称**：大气污染溯源分析系统

**技术栈**：FastAPI后端 + Vue 3前端

**核心功能**：基于ReAct架构的智能污染源分析

**端口**：后端8000，前端5174

## 2. 核心架构

### 2.1 ReAct Agent架构
系统采用ReAct（Reasoning + Acting）模式进行智能分析：
- 思考 → 行动 → 观察 → 循环
- 动态规划执行方案
- 工具注册表智能选择
- LLM自主决策，动态工具调用

### 2.2 多专家Agent系统
系统包含4个专业专家Agent：
- **WeatherExpertAgent**：气象数据分析（天气、轨迹、风向、上风向分析）
- **ComponentExpertAgent**：污染物组分分析（VOCs、PM2.5、PM10、PMF、OBM、OFP源解析）
- **VizExpertAgent**：数据可视化（图表生成、可视化分析，15种图表类型）
- **ReportExpertAgent**：综合报告生成（分析总结、结论建议）

**专家路由器**：智能任务路由、依赖图调度、健康感知路由、可观测性监控

### 2.3 双模式Agent架构
系统支持助手模式（办公任务）和专家模式（数据分析）两种工作模式，通过call_sub_agent工具实现双向互调。

## 3. 功能需求

### 3.1 数据查询功能
- 查询广东省城市日空气质量数据（新标准HJ 633-2026）
- 查询天气预报（未来7-16天）
- 查询VOCs、颗粒物组分数据
- 站点查询、上风向分析

### 3.2 数据分析功能
- PMF源解析
- OBM/OFP分析
- 智能图表生成（15种图表类型）
- 综合报告生成

### 3.3 办公辅助功能
- 文件操作（读取、编辑、写入、搜索）
- Office文档处理（Word、Excel、PPT）
- 图片分析
- 系统命令执行

### 3.4 通知与任务管理
- 微信通知（文本、图片、文件）
- 定时任务创建
- 后台任务执行（不阻塞对话）

### 3.5 空气质量查询功能
- 广东省城市日数据查询
- 统计报表查询
- 对比分析报告
- AQI日历热力图生成

## 4. 技术规范

### 4.1 数据格式规范
- **UDF v2.0**：数据分析工具全链路统一格式
- **Chart v3.1**：图表格式标准
- **字段映射系统**：260个字段标准化

### 4.2 工具架构规范
- **数据分析工具**：使用Context-Aware V2架构
- **办公助手工具**：使用简化格式
- **任务管理工具**：使用简化UDF v2.0

### 4.3 错误处理机制
- Input Adapter：参数规范化
- Reflexion：分析失败原因
- ReAsk：智能重试

## 5. 部署要求

### 5.1 后端部署
```bash
cd backend && python -m uvicorn app.main:app --reload
```

### 5.2 前端部署
```bash
cd frontend && npm install && npm run dev
```
