# 模板报告Prompt改进文档

## 改进日期
2026-01-28

## 改进目标
解决报告生成过程中的查询重复、字段不完整、数据粒度不匹配等问题

## 问题分析

### 问题1：查询重复和冗余
**现象**：同一时间段的数据被查询多次
- 查询1：广东省2025年全年数据 → 21条记录
- 查询2：广东省2025年全年各城市数据（同比） → 22条记录
- 查询3：广东省2025年全年各城市数据（详细） → 64条记录

**原因**：Prompt没有强制要求"一次性查询所有字段"

### 问题2：字段不完整
**现象**：需要多次查询补充缺失字段
- 第1次查询：只包含 AQI、PM2.5、O3
- 第2次查询：补充 超标天数
- 第3次查询：补充 综合指数、排名

**原因**：Prompt没有要求"分析模板需求，列出所有字段"

### 问题3：数据粒度不匹配
**现象**：请求日度数据，返回年度聚合数据
```
查询: "查询东莞市2025年的日度空气质量数据"
返回: 1条聚合记录（timestamp="2025-01-01~ 2025-12-31"）
```

**原因**：Prompt没有要求"明确指定预期记录数"

### 问题4：数据字段不一致
**现象**：不同查询返回的字段差异很大
- 查询1返回：PM2.5, O3_8h
- 查询2返回：AQI, PM2.5, O3_8h, CompositeIndex_Compare
- 查询3返回：AQI, PM2.5, O3_8h, OverDays

**原因**：没有统一的字段要求规范

## 改进方案

### 改进1：新增"字段完整性要求"（第123-144行）

**新增内容**：
```markdown
4. **字段完整性要求（强制）**：⭐⭐⭐ 最高优先级
   在生成每个查询question之前，必须完成以下步骤：

   a) **分析模板需求**：仔细阅读模板对应章节，列出该章节需要的所有数据字段
      例如：AQI达标率、PM2.5浓度、PM2.5同比变化、O3评价浓度、O3同比变化、
            超标天数、综合指数、综合指数排名等

   b) **生成完整question**：确保question中明确列出所有需要的字段，不遗漏任何一个
      ✅ 正确示例：
      "查询广东省2025年1月1日至2025年12月31日各城市空气质量数据，
       包括AQI达标率、PM2.5浓度、O3评价浓度、超标天数、综合指数、综合指数排名，
       并与2024年同期进行同比分析，返回PM2.5同比变化、O3同比变化、综合指数同比变化等字段"

      ❌ 错误示例：
      "查询广东省2025年空气质量数据"（字段不明确）
      "查询广东省2025年AQI、PM2.5数据"（遗漏其他字段）

   c) **一次性查询完整**：同一时间段、同一地区的所有数据必须在一个question中完整列出
      - 禁止分多次查询补充缺失字段
      - 禁止先查询部分字段，再查询其他字段
      - 如果模板需要10个字段，question必须明确列出全部10个字段

   d) **验证字段覆盖**：生成question后，检查是否覆盖了模板需要的所有字段
      - 如果模板表格有10列，question必须包含对应的10个字段
      - 如果模板需要同比数据，question必须明确要求返回同比字段
```

**预期效果**：
- ✅ 减少查询次数：从3-4次减少到1次
- ✅ 字段完整：一次性获取所有需要的字段
- ✅ 避免重复：同一数据不会被多次查询

### 改进2：新增"数据粒度明确要求"（第146-161行）

**新增内容**：
```markdown
5. **数据粒度明确要求（强制）**：⭐⭐⭐
   在question中必须明确指定数据粒度和预期记录数：

   - **年度数据**：明确写"年度汇总数据"或"按年统计"，并说明预期记录数
     ✅ 正确："查询广东省21个城市2025年的年度汇总数据（21条记录），
              包括全年AQI达标率、PM2.5年均浓度、全年超标天数等"

   - **月度数据**：明确写"月度数据"或"按月统计"，并说明预期记录数
     ✅ 正确："查询广州市2025年1-12月的月度数据（12条记录），
              包括每月AQI、PM2.5月均值等"

   - **日度数据**：明确写"每日数据"或"按日统计"，并说明预期记录数
     ✅ 正确："查询东莞市2025年的每日数据（365条记录），
              包括每天的AQI、PM2.5日均值、O3日最大8小时值等"
     ❌ 错误："查询东莞市2025年的日度空气质量数据"
              （没说明需要365条记录，API可能返回1条聚合数据）

   - **重要**：如果需要统计"超标天数"等指标，必须明确是要"每日数据"还是"年度汇总的超标天数字段"
     ✅ 方案1（推荐）："查询东莞市2025年的年度汇总数据（1条记录），
                       包括全年超标天数（OverDays字段）"
     ✅ 方案2："查询东莞市2025年的每日数据（365条记录），
              包括每天的AQI，用于统计超标天数"
```

**预期效果**：
- ✅ 数据粒度匹配：请求日度数据时，明确需要365条记录
- ✅ 避免歧义：API知道应该返回聚合数据还是明细数据
- ✅ 提高效率：优先使用API预计算的聚合字段（如OverDays）

## 改进前后对比

### 场景：查询广东省2025年各城市空气质量数据

#### 改进前
```
查询1: "查询广东省2025年空气质量数据"
返回: 21条记录（只有PM2.5、O3）

查询2: "查询广东省2025年各城市数据，包括超标天数"
返回: 22条记录（补充超标天数）

查询3: "查询广东省2025年各城市详细数据"
返回: 64条记录（补充综合指数、排名）

总计: 3次查询，107条记录（大量重复）
```

#### 改进后
```
查询1: "查询广东省21个城市2025年1月1日至2025年12月31日的年度汇总数据（21条记录），
       包括AQI达标率、PM2.5浓度、O3评价浓度、超标天数、综合指数、综合指数排名，
       并与2024年同期（2024-01-01至2024-12-31）进行同比分析，
       返回PM2.5同比变化、O3同比变化、综合指数同比变化等字段"
返回: 21条记录（包含所有需要的字段）

总计: 1次查询，21条记录
```

**改进效果**：
- 查询次数：3次 → 1次（减少67%）
- 数据量：107条 → 21条（减少80%）
- 字段完整性：需要3次补充 → 一次性完整

## 实施建议

### 1. 测试验证
建议使用相同的模板和时间范围，对比改进前后的查询行为：
- 记录查询次数
- 记录返回的字段列表
- 验证数据粒度是否匹配

### 2. 监控指标
- 平均查询次数（目标：≤2次）
- 字段完整率（目标：≥95%）
- 数据粒度匹配率（目标：100%）

### 3. 后续优化
如果仍然出现重复查询，可以考虑：
- 在Prompt中添加"查询去重检查"
- 在代码层面添加查询缓存机制
- 记录已查询的数据，避免重复请求
